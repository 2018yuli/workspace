## 常见问题

### 我的计划的使用限制是什么？

您可以发送的 Codex 消息数量取决于所使用的模型、编码任务的大小和复杂性，以及您是本地运行还是在云中运行它们。小脚本或常规功能可能仅消耗您配额的一小部分，而更大的代码库、长时间运行的任务或需要 Codex 保持更多上下文的扩展会话将每条消息使用更多。

GPT-5.5 使用的代币显著少于 GPT-5.4，以实现相当的结果。其 Codex 设置运行更快，并为大多数用户提供更高质量的结果。这些效率提升支持尽管 GPT-5.5 是一个显著更强大的模型，仍有慷慨的使用限制。

### 当您达到使用限制时会发生什么？

达到使用限制的 ChatGPT Plus 和 Pro 用户可以购买额外积分，以便继续工作，而无需升级现有计划。

具有 [灵活定价](https://help.openai.com/en/articles/11487671-flexible-pricing-for-the-enterprise-edu-and-business-plans) 的商业、教育和企业计划可以购买额外的工作区积分，以继续使用 Codex。

如果您接近使用限制，您也可以切换到更小的模型，以使您的使用限制持续更长时间。

所有用户也可以使用 API 密钥运行额外的本地任务，费用按 [标准 API 费率](https://platform.openai.com/docs/pricing) 收取。

### 图像生成如何计算使用限制？

图像生成计算在本地消息和云任务的 Codex 使用限制内。图像生成平均以 3-5 倍于没有图像生成的类似轮次的速度使用包含的限制，具体取决于图像质量和大小。在您达到包含的限制后，图像生成还会消耗 [积分](https://developers.openai.com/codex/pricing#credits-overview)。

免费计划不提供图像生成。当您使用 API 密钥使用 Codex 时，图像生成的费用适用于 API 定价，而不是包含的 ChatGPT 使用限制。

### 目前 Pro 的 Codex 使用促销是什么？

我们目前在两个 Pro 层级上提供额外的 Codex 使用量。

对于 **Pro $100**，为了庆祝发布，您将获得 **在 2026 年 5 月 31 日之前 2 倍的 Codex 使用量**。这意味着 10 倍的使用量，而不是标准的 5 倍。

**对于 Pro $200**，作为对我们最忠实客户的感谢，我们继续提供之前 2 倍促销的好处，这意味着 Pro $200 现在持续包含 20 倍的 Plus。此外，我们继续在有限时间内保持更高的 5 小时 Codex 限制，因此在 2026 年 5 月 31 日之前仍为 25 倍的 Plus，而不是标准的 20 倍。

### 我在哪里可以查看我的当前使用限制？

您可以在 [Codex 使用\\ 仪表板](https://chatgpt.com/codex/settings/usage) 中找到当前限制。如果您想在活动的 Codex CLI 会话期间查看剩余限制，可以使用 `/status`。

### 积分如何运作？

积分使您在达到包含的使用限制后继续使用 Codex。使用量按您使用的模型和功能消耗可用积分，使您能够毫无中断地继续工作。

截至 4 月 2 日，我们正在将定价迁移到基于 API 令牌的费率。积分仍然是客户购买和消耗的核心定价单位，但使用量基于消耗的代币，计算为工作区消耗的每百万输入代币、缓存输入代币和输出代币。了解代币的 [更多信息](https://help.openai.com/en/articles/4936856-what-are-tokens-and-how-to-count-them)。

此格式将您计划的每条消息的平均估算替换为代币使用量与积分之间的直接映射。当您希望更清晰地查看输入、缓存输入和输出如何影响积分消耗时，它特别有用。

### 什么被视为代码审查使用？

代码审查使用仅在通过 GitHub 运行 Codex 审查时应用——例如，当您在拉取请求中标记 `@Codex` 以进行审查或在您的存储库上启用自动审查时。局部或在 GitHub 之外运行的审查计入您的一般使用限制。

### 我可以做些什么来使我的使用限制更持久？

上述使用限制和积分是平均费率。您可以尝试以下提示来最大化您的限制：

- **控制提示的大小。** 用准确的指令给 Codex，但删除不必要的上下文。
- **减少 AGENTS.md 的大小。** 如果您在大型项目中工作，可以通过 [将其嵌套在您的存储库中](https://developers.openai.com/codex/guides/agents-md#layer-project-instructions)来控制通过 AGENTS.md 文件注入的上下文量。
- **限制您使用的 MCP 服务器数量。** 每个 [MCP](https://developers.openai.com/codex/mcp) 添加到 Codex 中会为您的消息增加更多上下文并消耗更多的限制。在不需要时禁用 MCP 服务器。
- **切换到小模型进行常规任务。** 使用 GPT-5.4 或 GPT-5.4-mini 可以根据您切换的模型延长本地消息使用限制。
