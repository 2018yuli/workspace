## 在交互模式下运行

Codex 启动为全屏终端 UI，可以读取您的代码库、进行编辑并在您一起迭代时运行命令。每当您想要一个可以实时查看 Codex 动作的对话式工作流时，请使用它。

```
codex
```

您还可以在命令行中指定初始提示。

```
codex "解释这个代码库给我听"
```

一旦会话打开，您可以：

- 将提示、代码片段或截图（见 [图像输入](https://developers.openai.com/codex/cli/features#image-inputs)）直接发送到编写器。
- 观看 Codex 在做出更改之前解释其计划，并在线批准或拒绝步骤。
- 在 TUI 中阅读语法高亮的 Markdown 代码块和差异，然后使用 `/theme` 预览并保存偏好的主题。
- 使用 `/clear` 清除终端并开始一次新的聊天，或按 `Ctrl` + `L` 清除屏幕而不启动新的对话。
- 使用 `/copy` 或按 `Ctrl` + `O` 复制最新的 Codex 输出。如果一个回合仍在进行，Codex 将复制最近完成的输出，而不是正在进行的文本。
- 在 Codex 正在运行时按 `Tab` 将后续文本、斜杠命令或 `!` shell 命令排队至下一个回合。
- 在编写器中使用 `Up`/ `Down` 导航草稿历史；Codex 会恢复先前草稿文本和图像占位符。
- 按 `Ctrl` + `R` 从编写器中搜索提示历史，然后按 `Enter` 接受匹配或按 `Esc` 取消。
- 按 `Ctrl` + `C` 或使用 `/exit` 在完成时关闭交互会话。

## 恢复对话

Codex 将您的记录保存在本地，因此您可以从上次停下的地方继续，而无需重复上下文。当您想使用相同的代码库状态和指令重新打开先前线程时，请使用 `resume` 子命令。

- `codex resume` 启动最近交互会话的选择器。高亮显示一轮以查看其摘要，然后按 `Enter` 重新打开它。
- `codex resume --all` 显示当前工作目录之外的会话，因此您可以重新打开任何本地运行。
- `codex resume --last` 跳过选择器，直接跳转到当前工作目录的最近会话（添加 `--all` 以忽略当前工作目录过滤）。
- `codex resume <SESSION_ID>` 指定特定的运行。您可以从选择器、`/status` 或 `~/.codex/sessions/` 下的文件复制 ID。

非交互自动化运行也可以恢复：

```
codex exec resume --last "修复您发现的竞争条件"
codex exec resume 7f9f9a2e-1b3c-4c7a-9b0e-.... "实施计划"
```

每个恢复的运行都保留原始记录、计划历史和批准，因此 Codex 可以在您提供新指令时利用先前上下文。如果您需要在恢复之前引导环境，请使用 `--cd` 重写工作目录或使用 `--add-dir` 添加额外根目录。

## 将 TUI 连接到远程应用服务器

远程 TUI 模式允许您在一台机器上运行 Codex 应用服务器，并从另一台机器使用 Codex 终端 UI。这在代码、凭证或执行环境位于远程主机上，但您想要本地交互式 TUI 体验时非常有用。

在应拥有工作区并运行命令的机器上启动应用服务器：

```
codex app-server --listen ws://127.0.0.1:4500
```

然后从运行 TUI 的机器连接：

```
codex --remote ws://127.0.0.1:4500
```

要从另一台机器访问，请将应用服务器绑定到可访问的接口，例如：

```
codex app-server --listen ws://0.0.0.0:4500
```

`--remote` 仅接受显式的 `ws://host:port` 和 `wss://host:port` 地址。对于普通的 WebSocket 连接，优先使用本地主机地址或 SSH 端口转发。如果您在本地主机之外公开侦听器，在真正的远程使用之前配置身份验证并将经过身份验证的非本地连接放置在 TLS 后面。

Codex 支持这些用于远程 TUI 连接的 WebSocket 身份验证模式：

- **无 WebSocket 身份验证**：最佳用于本地主机侦听器或 SSH 端口转发连接。Codex 可以启动无需身份验证的非本地侦听器，但会记录警告，启动横幅提醒您在真正的远程使用之前配置身份验证。
- **能力令牌**：在应用服务器主机上存储共享令牌的文件，使用 `--ws-auth capability-token --ws-token-file /abs/path/to/token` 启动服务器，然后在 TUI 主机上的环境变量中设置相同的令牌，并通过 `--remote-auth-token-env <ENV_VAR>` 传递。
- **签名承载令牌**：在应用服务器主机上存储 HMAC 共享密钥的文件，使用 `--ws-auth signed-bearer-token --ws-shared-secret-file /abs/path/to/secret` 启动服务器，并让 TUI 通过 `--remote-auth-token-env <ENV_VAR>` 发送签名的 JWT 承载令牌。共享密钥必须至少为 32 字节。签名令牌使用 HS256，必须包括 `exp`；当这些声明或服务器选项存在时，Codex 还会验证 `nbf`、`iss` 和 `aud`。

要在应用服务器主机上创建能力令牌，请生成一个随机令牌文件，权限仅授予您的用户读取：

```
TOKEN_FILE="$HOME/.codex/codex-app-server-token"
install -d -m 700 "$(dirname "$TOKEN_FILE")"
openssl rand -base64 32 > "$TOKEN_FILE"
chmod 600 "$TOKEN_FILE"
```

将令牌文件视作密码，如果泄露则重新生成。

然后使用该令牌文件启动应用服务器。例如，在 TLS 代理后面的能力令牌：

```
# 远程主机
TOKEN_FILE="$HOME/.codex/codex-app-server-token"
codex app-server \
  --listen ws://0.0.0.0:4500 \
  --ws-auth capability-token \
  --ws-token-file "$TOKEN_FILE"

# TUI 主机
export CODEX_REMOTE_AUTH_TOKEN="$(ssh devbox 'cat ~/.codex/codex-app-server-token')"
codex --remote wss://codex-devbox.example.com:4500 \
  --remote-auth-token-env CODEX_REMOTE_AUTH_TOKEN
```

TUI 在 WebSocket 握手期间将远程身份验证令牌发送为 `Authorization: Bearer <token>`。Codex 仅在 `wss://` URL 或主机为 `localhost`、`127.0.0.1` 或 `::1` 的 `ws://` URL 上发送这些令牌，因此在客户端需要通过网络进行身份验证时，请将非本地远程侦听器放置在 TLS 后面。

## 模型和推理

对于 Codex 中的大多数任务，当可用时推荐使用 `gpt-5.5` 模型。这是 OpenAI 用于复杂编码、计算机使用、知识工作和研究工作流的最新前沿模型，具有更强的计划、工具使用和对多步骤任务的跟进能力。如果 `gpt-5.5` 尚不可用，请继续使用 `gpt-5.4`。对于额外快速的任务，ChatGPT Pro 订阅者可以访问 GPT-5.3-Codex-Spark 模型，处于研究预览状态。

使用 `/model` 命令在会话期间切换模型，或在启动 CLI 时指定一个。

```
codex --model gpt-5.5
```

[了解 Codex 中可用的模型](https://developers.openai.com/codex/models)。

## 功能标志

Codex 包含一小组功能标志。使用 `features` 子命令检查可用项并在您的配置中持久化更改。

```
codex features list
codex features enable unified_exec
codex features disable shell_snapshot
```

`codex features enable <feature>` 和 `codex features disable <feature>` 将写入 `~/.codex/config.toml`。如果您使用 `--profile` 启动 Codex，则 Codex 会将更改存储在该配置文件中而不是根配置中。

## 子代理

使用 Codex 子代理工作流来并行化更大的任务。有关设置、角色配置（`config.toml` 中的 `[agents]`）和示例，请参见 [子代理](https://developers.openai.com/codex/subagents)。

Codex 仅在您明确要求时生成子代理。由于每个子代理都会执行自己的模型和工具工作，因此子代理工作流消耗的令牌比可比的单代理运行更多。

## 图像输入

附加屏幕截图或设计规格，以便 Codex 可以在您的提示旁边读取图像细节。您可以将图像粘贴到交互式编写器中或在命令行中提供文件。

```
codex -i screenshot.png "解释这个错误"
```

```
codex --image img1.png,img2.jpg "总结这些图表"
```

Codex 接受 PNG 和 JPEG 等常见格式。使用用逗号分隔的文件名表示两个或更多图像，并将它们与文本说明结合以添加上下文。

## 图像生成

请 Codex 直接在 CLI 中生成或编辑图像。这适用于如图标、横幅、插图、精灵图和占位符艺术等资产。如果您希望 Codex 转换或扩展现有资产，请在提示中附上参考图像。

您可以用自然语言提问或通过在提示中包含 `$imagegen` 显式调用图像生成技能。

内置图像生成使用 `gpt-image-2`，计算在您一般的 Codex 使用限制内，并且根据图像质量和大小，平均使用3-5倍速度比没有图像生成的类似回合更快。有关详细信息，请参见 [定价](https://developers.openai.com/codex/pricing#image-generation-usage-limits)。有关提示技巧和模型详细信息，请参见 [图像生成指南](https://developers.openai.com/api/docs/guides/image-generation)。

对于更大的图像生成批次，请在环境变量中设置 `OPENAI_API_KEY` 并请求 Codex 通过 API 生成图像，以便应用 API 定价。

## 语法高亮和主题

TUI 对带框的 Markdown 代码块和文件差异进行语法高亮，使代码在审查和调试时更易于扫描。

使用 `/theme` 打开主题选择器，实时预览主题并将选定的主题保存到 `~/.codex/config.toml` 中。您还可以在 `$CODEX_HOME/themes` 下添加自定义 `.tmTheme` 文件并在选择器中选择它们。

## 运行本地代码审查

在 CLI 中输入 `/review` 打开 Codex 的审查预设。CLI 启动专用审查器，读取您选择的差异并报告优先级高、可操作的发现，而不会触及您的工作树。默认情况下，它使用当前会话模型；在 `config.toml` 中设置 `review_model` 来覆盖。

- **基于基础分支的审查** 让您选择一个本地分支；Codex 会寻找它的上游合并基线，检查您的工作差异，并在您打开拉取请求之前突出显示最大风险。
- **审查未提交的更改** 检查所有已暂存、未暂存或未跟踪的内容，以便在提交之前解决问题。
- **审查一次提交** 列出最近的提交，并让 Codex 读取您选择的 SHA 的确切变更集。
- **自定义审查说明** 接受您自己的措辞（例如，“专注于可访问性回归”）并使用该提示运行相同的审查器。

每次运行作为其在记录中的一个回合显示，因此您可以在代码演变时重新运行审查并比较反馈。

## 网络搜索

Codex 附带一个第一方网络搜索工具。对于 Codex CLI 中的本地任务，Codex 默认启用网络搜索，并从网络搜索缓存提供结果。缓存是 OpenAI 维护的网络结果索引，因此缓存模式返回预索引的结果，而不是获取实时页面。这减少了从任意实时内容引入提示注入的暴露，但您仍应将网络结果视为不受信任。如果您使用 `--yolo` 或其他 [完全访问沙盒设置](https://developers.openai.com/codex/agent-approvals-security)，网络搜索默认使用实时结果。要获取最新数据，请在单次运行时传递 `--search`，或在 [配置基础](https://developers.openai.com/codex/config-basic) 中设置 `web_search = "live"`。您还可以设置 `web_search = "disabled"` 来关闭该工具。

每当 Codex 查找某些内容时，您会在记录或 `codex exec --json` 输出中看到 `web_search` 项目。

## 使用输入提示运行

当您只需要一个快速答案时，使用单个提示运行 Codex，跳过交互 UI。

```
codex "解释这个代码库"
```

Codex 将读取工作目录，制定计划，并在退出之前将响应流回到您的终端。使用类似 `--path` 的标志来针对特定目录，或使用 `--model` 预先指定行为。

## Shell 完成

通过为您的 shell 安装生成的完成脚本来加快日常使用：

```
codex completion bash
codex completion zsh
codex completion fish
```

在您的 shell 配置文件中运行完成脚本，以便为新会话设置完成。例如，如果您使用 `zsh`，可以将以下内容添加到 `~/.zshrc` 文件的末尾：

```
# ~/.zshrc
eval "$(codex completion zsh)"
```

启动新会话，输入 `codex`，然后按 `Tab` 以查看完成。如果您看到 `command not found: compdef` 错误，请在 `~/.zshrc` 文件中的 `eval "$(codex completion zsh)"` 行之前添加 `autoload -Uz compinit && compinit`，然后重启您的 shell。

## 批准模式

批准模式定义了 Codex 可以在不停止确认的情况下做多少事情。在交互会话中使用 `/permissions` 根据您的舒适度更改模式。

- **自动**（默认）允许 Codex 读取文件、编辑和运行工作目录中的命令。在触及该范围之外的任何内容或使用网络之前，仍会进行询问。
- **只读** 将 Codex 保持在咨询模式。它可以浏览文件，但不会进行更改或运行命令，直到您批准计划。
- **完全访问** 授予 Codex 在您的计算机上工作的能力，包括网络访问，无需询问。请谨慎使用，仅在您信任代码库和任务时使用。

Codex 总是展示其操作的记录，因此您可以使用常规 git 工作流程审查或回滚更改。

## 脚本化 Codex

通过 `exec` 子命令自动化工作流或将 Codex 接入现有脚本。这会以非交互方式运行 Codex，将最终计划和结果通过 `stdout` 输出。

```
codex exec "修复 CI 失败"
```

将 `exec` 与 shell 脚本组合以构建自定义工作流，例如自动更新变更日志、对问题进行排序或在 PR 发布之前强制进行编辑检查。

## 使用 Codex 云

`codex cloud` 命令使您能够在不离开终端的情况下处理和启动 [Codex 云任务](https://developers.openai.com/codex/cloud)。不带参数运行它以打开交互选择器，浏览活动或完成的任务，并将更改应用于本地项目。

您还可以直接从终端启动任务：

```
codex cloud exec --env ENV_ID "总结未解决的错误"
```

添加 `--attempts`（1–4）以在希望 Codex 云生成多个解决方案时请求最佳的 N 次运行。例如，`codex cloud exec --env ENV_ID --attempts 3 "总结未解决的错误"`。

环境 ID 来自您的 Codex 云配置—使用 `codex cloud`，然后按 `Ctrl` + `O` 选择环境或通过 Web 仪表板确认确切值。身份验证遵循您现有的 CLI 登录，如果提交失败，该命令将以非零状态退出，因此您可以将其接入脚本或 CI。

## 斜杠命令

斜杠命令使您能够快速访问特定工作流，如 `/review`、`/fork` 或您自己可重用的提示。Codex 附带一组经过策划的内置命令，您可以为团队特定任务或个人快捷方式创建自定义命令。

请参见 [斜杠命令指南](https://developers.openai.com/codex/guides/slash-commands)，以浏览内置命令目录，了解如何撰写自定义命令，并了解它们在磁盘上的位置。

## 提示编辑器

当您正在撰写较长的提示时，切换到完整编辑器并将结果发送回编写器可能更容易。

在提示输入中，按 `Ctrl` + `G` 打开由 `VISUAL` 环境变量定义的编辑器（如果未设置 `VISUAL`，则使用 `EDITOR`）。

## 模型上下文协议 (MCP)

通过配置模型上下文协议服务器，将 Codex 连接到更多工具。在 `~/.codex/config.toml` 中添加 STDIO 或流式 HTTP 服务器，或使用 `codex mcp` CLI 命令管理它们—Codex 在会话启动时会自动启动它们，并将其工具暴露在内置工具旁边。当您需要它在另一个代理内部时，甚至可以将 Codex 本身作为 MCP 服务器运行。

请参阅 [模型上下文协议](https://developers.openai.com/codex/mcp) 以获取示例配置、支持的身份验证流和更详细的指南。

## 提示和快捷方式

- 在编写器中输入 `@` 以打开整个工作区根目录的模糊文件搜索；按 `Tab` 或 `Enter` 将高亮路径放入您的消息中。
- 在 Codex 运行时按 `Enter` 将新指令注入到当前回合，或按 `Tab` 将后续输入排队至下一个回合。排队的输入可以是常规提示、斜杠命令如 `/review`，或 `!` shell 命令。Codex 在运行时解析排队的斜杠命令。
- 在行前缀用 `!` 运行本地 shell 命令（例如，`!ls`）。Codex 将输出视为用户提供的命令结果，并且仍然应用您的批准和沙盒设置。
- 当编写器为空时，双击 `Esc` 以编辑您之前的用户消息。继续按 `Esc` 向后浏览记录，然后按 `Enter` 从该点分叉。
- 使用 `codex --cd <path>` 从任何目录启动 Codex，以在不先运行 `cd` 的情况下设置工作根目录。活动路径出现在 TUI 标头中。
- 使用 `--add-dir` 公开更多可写根（例如，`codex --cd apps/frontend --add-dir ../backend --add-dir ../shared`），当您需要在多个项目之间协调更改时。
- 确保在启动 Codex 之前环境已经设置好，以免它在探测要激活的内容时消耗令牌。例如，提前源您的 Python 虚拟环境（或其他语言环境）、启动任何所需的守护进程，并导出您预期使用的环境变量。
