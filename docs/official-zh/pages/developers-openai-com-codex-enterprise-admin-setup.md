## 企业级安全和隐私

Codex 支持 ChatGPT 企业安全功能，包括：

- 不对企业数据进行训练
- 应用、CLI 和 IDE 的零数据保留（代码保留在开发者环境中）
- 遵循 ChatGPT 企业政策的居住和保留
- 细粒度用户访问控制
- 静态数据加密（AES-256）和传输中加密（TLS 1.2+）
- 通过 ChatGPT 合规性 API 进行审计日志

有关安全控制和运行时保护，请参阅[代理批准与安全](https://developers.openai.com/codex/agent-approvals-security)。有关更多详细信息，请参阅[零数据保留 (ZDR)](https://platform.openai.com/docs/guides/your-data#zero-data-retention)。有关更广泛的企业安全概览，请参阅[Codex 安全白皮书](https://trust.openai.com/?itemUid=382f924d-54f3-43a8-a9df-c39e6c959958&source=click)。

## 先决条件：确定负责人和推出策略

在推出过程中，团队成员可能会支持将 Codex 集成到您的组织的不同方面。确保您有以下负责人：

- **ChatGPT 企业工作区所有者：** 必须配置工作区中的 Codex 设置。
- **安全负责人：** 确定 Codex 的代理权限设置。
- **分析负责人：** 将分析和合规 API 集成到您的数据管道中。

决定您将使用哪些 Codex 界面：

- **Codex 本地：** 包括 Codex 应用、CLI 和 IDE 扩展。代理在开发者的计算机的沙盒中运行。
- **Codex 云：** 包括托管的 Codex 功能（包括 Codex 云、iOS、代码审查和由[Slack 集成](https://developers.openai.com/codex/integrations/slack)或[Linear 集成](https://developers.openai.com/codex/integrations/linear)创建的任务）。代理在托管的容器中远程运行，使用您的代码库。
- **两者均可：** 同时使用本地 + 云。

您可以启用本地、云或两者，并通过工作区设置和基于角色的访问控制（RBAC）控制访问。

## 第一步：在工作区中启用 Codex

您可以在 ChatGPT 企业工作区设置中配置对 Codex 的访问。

转到 [工作区设置 > 设置和权限](https://chatgpt.com/admin/settings)。

### Codex 本地

Codex 本地在新的 ChatGPT 企业工作区中默认启用。如果您不是 ChatGPT 工作区的所有者，您可以通过[安装 Codex](https://developers.openai.com/codex/quickstart)并使用您的工作电子邮件登录来测试您是否有访问权限。

开启 **允许成员使用 Codex Local**。

这允许允许的用户使用 Codex 应用、CLI 和 IDE 扩展。

如果此切换关闭，尝试使用 Codex 应用、CLI 或 IDE 的用户将看到以下错误：“403 - 未授权。请联系您的 ChatGPT 管理员以获取访问权限。”

#### 为 Codex CLI 启用设备代码身份验证

允许开发人员在非交互环境（例如远程开发箱）中使用设备代码登录 Codex CLI。更多详细信息请参见[身份验证](https://developers.openai.com/codex/auth/)。

![Codex 本地切换](https://developers.openai.com/images/codex/enterprise/local-toggle-config.png)

### Codex 云

### 先决条件

Codex 云需要 **GitHub（云托管）仓库**。如果您的代码库在本地或不在 GitHub 上，您可以使用 Codex SDK 在您自己的基础设施上构建类似的工作流程。

要作为管理员设置 Codex，您必须对组织中常用的存储库具有 GitHub 访问权限。如果您没有必要的访问权限，请与您的工程团队中有权限的人合作。

### 在工作区设置中启用 Codex 云

首先在 [工作区设置 > 设置和权限](https://chatgpt.com/admin/settings)的 Codex 部分中开启 ChatGPT GitHub 连接器。

要为您的工作区启用 Codex 云，请开启 **允许成员使用 Codex 云**。一旦启用，用户可以直接从 ChatGPT 左侧导航面板访问 Codex。

请注意，Codex 可能需要最多 10 分钟才能出现在 ChatGPT 中。

#### 启用 Codex Slack 应用以在任务完成时发布答案

当任务完成时，Codex 会将其完整答案发布回 Slack。否则，Codex 仅发布链接到任务。

要了解更多，请参见 [Codex 在 Slack 中](https://developers.openai.com/codex/integrations/slack)。

#### 启用 Codex 代理访问互联网

默认情况下，Codex 云代理在运行时没有互联网访问权限，以帮助防止诸如提示注入的安全和安全风险。

此设置允许用户使用白名单用于常见软件依赖域，添加域和受信任网站，并指定允许的 HTTP 方法。

有关互联网访问和运行时控制的安全影响，请参见[代理批准与安全](https://developers.openai.com/codex/agent-approvals-security)。

![Codex 云切换](https://developers.openai.com/images/codex/enterprise/cloud-toggle-config.png)

## 第二步：设置自定义角色（RBAC）

使用 RBAC 控制对 Codex 本地和 Codex 云的细粒度权限。

![Codex 云切换](https://developers.openai.com/images/codex/enterprise/rbac_custom_roles.png)

### RBAC 允许您执行的操作

工作区管理员可以在 ChatGPT 管理设置中使用 RBAC：

- 设置未分配自定义角色的用户的默认角色
- 创建具有细粒度权限的自定义角色
- 将一个或多个自定义角色分配给组
- 通过 SCIM 自动同步用户到组
- 从自定义角色标签集中集中管理角色

用户可以继承多个角色，并且权限会解析为这些角色之间最宽松（限制最少）的访问。

### 创建 Codex 管理组

设置专门的“Codex 管理”组，而不是将 Codex 管理权限授予广泛的受众。

**允许成员管理 Codex** 切换将授予 Codex 管理角色。Codex 管理者可以：

- 查看 Codex [工作区分析](https://chatgpt.com/codex/settings/analytics)
- 打开 Codex [政策页面](https://chatgpt.com/codex/settings/policies)以管理云托管的 `requirements.toml` 政策
- 将那些管理政策分配给用户组或配置默认的后备政策
- 管理 Codex 云环境，包括编辑和删除环境

将此角色用于少数负责 Codex 推出、政策管理和治理的管理员。一般 Codex 用户不需要此角色。您无需 Codex 云就可以启用此切换。

推荐的推出模式：

- 创建一个“Codex 用户”组，供应使用 Codex 的人员
- 创建一个单独的“Codex 管理”组，供少数负责管理 Codex 设置和政策的人员
- 将自定义角色分配给仅“Codex 管理”组，允许成员管理 Codex 的切换为启用
- 将“Codex 管理”组的成员资格限制为工作区所有者或指定的平台、IT 和治理操作员
- 如果您使用 SCIM，请使用您的身份提供者支持“Codex 管理”组，以便成员资格更改可审计并集中管理

这种分离使推出 Codex 更加容易，同时保持分析、环境管理和政策部署限制在可信管理员之间。有关 RBAC 设置的详细信息及完整权限模型，请参见 [OpenAI RBAC 帮助中心文章](https://help.openai.com/en/articles/11750701-rbac)。

## 第三步：配置 Codex 本地要求

Codex 管理员可以从 Codex [政策页面](https://chatgpt.com/codex/settings/policies)部署管理员强制的 `requirements.toml` 政策。

当您希望对不同组应用不同的本地 Codex 限制，而无需先分发设备级文件时，请使用此页面。托管政策使用与[托管配置](https://developers.openai.com/codex/enterprise/managed-configuration)中描述的相同 `requirements.toml` 格式，因此您可以定义允许的批准政策、沙盒模式、网页搜索行为、MCP 服务器白名单、功能固定和限制命令规则。要禁用浏览器使用、应用内浏览器或计算机使用，请参见 [禁用 Codex 特性界面](https://developers.openai.com/codex/enterprise/managed-configuration#disable-codex-feature-surfaces)。

![Codex 政策和配置页面](https://developers.openai.com/images/codex/enterprise/policies_and_configurations_page.png)

推荐的设置：

1. 为大多数用户创建基线政策，然后仅在需要时创建更严格或更宽松的变体。
2. 将每个管理政策分配给特定用户组，并为其他所有用户配置默认的后备政策。
3. 小心排序组规则。如果用户匹配多个组特定规则，则应用第一个匹配规则。
4. 将每个政策视为该组的完整配置文件。Codex 不会从后续匹配的组规则中填写缺失字段。

这些云托管政策在用户使用 ChatGPT 登录时适用于 Codex 本地界面，包括 Codex 应用、CLI 和 IDE 扩展。

### 示例 requirements.toml 政策

使用云托管的 `requirements.toml` 政策为每个组强制执行您想要的保护措施。下面的片段是您可以调整的示例，而不是必需的设置。

![示例受管要求政策](https://developers.openai.com/images/codex/enterprise/example_policy.png)

示例：限制网页搜索、沙盒模式和标准本地推出的批准：

```
allowed_web_search_modes = ["disabled", "cached"]
allowed_sandbox_modes = ["workspace-write"]
allowed_approval_policies = ["on-request"]

```

示例：添加限制命令规则，当您希望管理员阻止或限制特定命令时：

```
[rules]
prefix_rules = [\
  { pattern = [{ token = "git" }, { any_of = ["push", "commit"] }], decision = "prompt", justification = "Require review before mutating remote history." },\
]

```

您可以单独使用任意示例，或将它们组合在一个组的单个管理政策中。有关确切的键、优先级和更多示例，请参见[托管配置](https://developers.openai.com/codex/enterprise/managed-configuration)和[代理批准与安全](https://developers.openai.com/codex/agent-approvals-security)。

### 检查用户政策

使用工作流程末尾的政策查找工具确认适用于用户的管理政策。您可以通过组检查政策分配或通过输入用户电子邮件进行检查。

![通过组或用户电子邮件查找政策](https://developers.openai.com/images/codex/enterprise/policy_lookup.png)

如果您计划限制登录方法或本地客户端的工作区，请参阅[身份验证](https://developers.openai.com/codex/auth)中的管理员管理的身份验证限制。

## 第四步：通过团队配置标准化本地配置

希望在整个组织中标准化 Codex 的团队可以使用团队配置共享默认值、规则和技能，而无需在每个本地配置上重复设置。

您可以将团队配置设置检查到 `.codex` 目录下的存储库中。当用户打开该存储库时，Codex 会自动获取团队配置设置。

首先在您流量最高的存储库中使用团队配置，以便团队在使用 Codex 最多的地方获得一致的行为。

| 类型 | 路径 | 用于 |
| --- | --- | --- |
| [配置基础](https://developers.openai.com/codex/config-basic) | `config.toml` | 设置沙盒模式、批准、模型、推理努力等的默认值。 |
| [规则](https://developers.openai.com/codex/rules) | `rules/` | 控制 Codex 可以在沙盒外运行的命令。 |
| [技能](https://developers.openai.com/codex/skills) | `skills/` | 使共享技能可供您的团队使用。 |

有关位置和优先级，请参见[配置基础](https://developers.openai.com/codex/config-basic#configuration-precedence)。

## 第五步：配置 Codex 云使用（如果启用）

此步骤介绍在您启用 Codex 云工作区切换后，仓库和环境的设置。

### 将 Codex 云连接到仓库

1. 导航到 [Codex](https://chatgpt.com/codex) 并选择 **开始使用**
2. 选择 **连接到 GitHub** 以安装 ChatGPT GitHub 连接器，如果您尚未将 GitHub 连接到 ChatGPT
3. 安装或连接 ChatGPT GitHub 连接器
4. 选择 ChatGPT 连接器的安装目标（通常是您的主要组织）
5. 允许您希望连接到 Codex 的仓库

对于 GitHub 企业管理用户（EMU），组织所有者必须先为组织安装 Codex GitHub 应用，然后用户才能在 Codex 云中连接存储库。

有关更多信息，请参见[云环境](https://developers.openai.com/codex/cloud/environments)。

Codex 使用短期、最小权限的 GitHub 应用安装令牌进行每个操作，并尊重用户现有的 GitHub 仓库权限和分支保护规则。

### 配置 IP 地址

如果您的 GitHub 组织控制应用程序连接所使用的 IP 地址，请确保包括这些[出口 IP 范围](https://openai.com/chatgpt-agents.json)。

这些 IP 范围可能会发生变化。考虑自动检查它们并根据最新值更新您的白名单。

### 启用 Codex 云的代码审查

要允许 Codex 在 GitHub 上执行代码审查，请转到 [设置 → 代码审查](https://chatgpt.com/codex/settings/code-review)。

您可以在仓库级别配置代码审查。用户还可以为他们的 PR 启用自动审查并选择 Codex 何时自动触发审查。有关更多详细信息，请访问[GitHub 集成页面](https://developers.openai.com/codex/integrations/github)。

使用概览页面确认您的工作区已启用代码审查，并查看可用的审查控制。

![代码审查设置概览](https://developers.openai.com/images/codex/enterprise/code_review_settings_overview.png)

使用自动审查设置决定 Codex 是否应为连接的仓库自动审查拉取请求。

![自动代码审查设置](https://developers.openai.com/images/codex/enterprise/auto_code_review_settings.png)

使用审查触发器控制哪些拉取请求事件应该开始 Codex 审查。

![代码审查触发器设置](https://developers.openai.com/images/codex/enterprise/review_triggers.png)

### 配置 Codex 安全

Codex 安全帮助工程和安全团队查找、确认和修复连接的 GitHub 仓库中的可能漏洞。

总体而言，Codex 安全：

- 按提交逐步扫描连接的存储库
- 排名可能的发现并在可能的情况下确认
- 以结构化发现的形式显示证据、重要性及建议的补救措施
- 允许团队优化仓库威胁模型以提高优先级和审查质量

有关设置、扫描创建、发现审查和威胁模型指导的信息，请参见[Codex 安全设置](https://developers.openai.com/codex/security/setup)。有关产品概述，请参见[Codex 安全](https://developers.openai.com/codex/security)。

集成文档还可用于[Slack](https://developers.openai.com/codex/integrations/slack)、[GitHub](https://developers.openai.com/codex/integrations/github)和[Linear](https://developers.openai.com/codex/integrations/linear)。

## 第六步：设置治理和可观察性

Codex 为企业团队提供了了解采用情况和影响的选项。尽早建立治理，以便您的团队能够跟踪采用情况、调查问题并支持合规工作流程。

### Codex 治理通常使用

- 分析仪表板以快速、自助式可见性
- 分析 API 用于程序化报告和业务智能集成
- 合规 API 用于审计和调查工作流程

### 推荐的基线设置

- 指定负责采用报告的负责人
- 指定负责审计和合规审查的负责人
- 定义审查节奏
- 决定成功的标准

### 分析 API 设置步骤

要设置分析 API 密钥：

1. 使用所有者或管理员身份登录 [OpenAI API 平台门户](https://platform.openai.com/)，并选择正确的组织。
2. 转到 [API 密钥页面](https://platform.openai.com/settings/organization/api-keys)。
3. 创建一个新的专用于 Codex Analytics 的秘密密钥，并给予描述性名称，例如 Codex Analytics API。
4. 为您的组织选择适当的项目。如果您只有一个项目，则默认项目即可。
5. 将密钥权限设置为仅可读取，因为该 API 仅检索分析数据。
6. 复制密钥值并安全存储，因为您只能查看一次。
7. 发送电子邮件至 [support@openai.com](mailto:support@openai.com)，请求将该密钥范围设置为 `codex.enterprise.analytics.read`。等待 OpenAI 确认您的 API 密钥已获得 Codex Analytics API 访问权限。

![Codex 分析密钥创建](https://developers.openai.com/images/codex/codex_analytics_key.png)

要使用分析 API 密钥：

1. 在 [ChatGPT 管理控制台](https://chatgpt.com/admin)的工作区详细信息下找到您的 `workspace_id`。
2. 使用您的平台 API 密钥调用分析 API，路径中包含您的 `workspace_id`，例如 `https://api.chatgpt.com/v1/analytics/codex`。
3. 选择您要查询的端点：

- /workspaces/`{workspace_id}`/usage
- /workspaces/`{workspace_id}`/code\_reviews
- /workspaces/`{workspace_id}`/code\_review\_responses

4. 如有需要，使用 `start_time` 和 `end_time` 设置报告日期范围。
5. 如果响应跨越多页，则使用 `next_page` 检索下一页结果。

示例 curl 命令以检索工作区使用情况：

```
curl -H "Authorization: Bearer YOUR_PLATFORM_API_KEY" \
  "https://api.chatgpt.com/v1/analytics/codex/workspaces/WORKSPACE_ID/usage"

```

有关分析 API 的更多详细信息，请参见[分析 API](https://developers.openai.com/codex/enterprise/governance#analytics-api)。

### 合规 API 设置步骤

要设置合规 API 密钥：

1. 使用所有者或管理员身份登录 [OpenAI API 平台门户](https://platform.openai.com/)，并选择正确的组织。
2. 转到 [API 密钥页面](https://platform.openai.com/settings/organization/api-keys)。
3. 创建一个新的专用于合规 API 的秘密密钥，并选择适当的项目。如果您只有一个项目，则默认项目即可。
4. 选择所有权限。
5. 复制密钥值并安全存储，因为您只能查看一次。
6. 发送电子邮件至 [support@openai.com](mailto:support@openai.com)，内容包括：

- API 密钥的最后 4 个数字
- 密钥名称
- 创建者名称
- 所需的范围：`read`、`delete` 或两者

7. 等待 OpenAI 确认您的 API 密钥已获得合规 API 访问权限。

要使用合规 API 密钥：

1. 在 [ChatGPT 管理控制台](https://chatgpt.com/admin)的工作区详细信息下找到您的 `workspace_id`。
2. 使用合规 API 访问 `https://api.chatgpt.com/v1/`
3. 在授权头中以 Bearer 令牌的形式传递您的合规 API 密钥。
4. 有关 Codex 相关的合规数据，使用以下端点：

- /compliance/workspaces/`{workspace_id}`/logs
- /compliance/workspaces/`{workspace_id}`/logs/`{log_file_id}`
- /compliance/workspaces/`{workspace_id}`/codex\_tasks
- /compliance/workspaces/`{workspace_id}`/codex\_environments

5. 对于大多数 Codex 合规集成，从日志端点开始，请请求 Codex 事件类型，如 CODEX\_LOG 或 CODEX\_SECURITY\_LOG。
6. 使用 /logs 列出可用的 Codex 合规日志文件，然后使用 /logs/`{log_file_id}` 下载特定文件。

示例 curl 命令列出合规日志文件：

```
curl -L -H "Authorization: Bearer YOUR_COMPLIANCE_API_KEY" \
  "https://api.chatgpt.com/v1/compliance/workspaces/WORKSPACE_ID/logs?event_type=CODEX_LOG&after=2026-03-01T00:00:00Z"

```

示例 curl 命令列出 Codex 任务：

```
curl -H "Authorization: Bearer YOUR_COMPLIANCE_API_KEY" \
  "https://api.chatgpt.com/v1/compliance/workspaces/WORKSPACE_ID/codex_tasks"

```

有关合规 API 的更多详细信息，请参见[合规 API](https://developers.openai.com/codex/enterprise/governance#compliance-api)。
