自定义提示已被弃用。请使用 [技能](https://developers.openai.com/codex/skills) 用于 Codex 可以显式或隐式调用的可重用指令。

自定义提示（已弃用）允许您将 Markdown 文件转换为可重用的提示，您可以在 Codex CLI 和 Codex IDE 扩展中作为斜杠命令调用。

自定义提示需要显式调用并位于您的本地 Codex 主目录（例如，`~/.codex`），因此它们不会通过您的代码库共享。如果您想分享一个提示（或希望 Codex 隐式调用它），请 [使用技能](https://developers.openai.com/codex/skills)。

1. 创建提示目录：

```
mkdir -p ~/.codex/prompts
```

2. 使用可重用指导创建 `~/.codex/prompts/draftpr.md`：

```
   ---
description: 准备一个分支、提交并打开一个草稿 PR
argument-hint: [FILES=<paths>] [PR_TITLE="<title>"]
   ---

为此工作创建一个名为 `dev/<feature_name>` 的分支。
如果指定了文件，首先暂存它们： $FILES。
使用清晰的信息提交暂存的更改。
在同一分支上打开一个草稿 PR。如果提供了 $PR_TITLE，请使用它；否则自己写一个简明的摘要。
```

3. 重新启动 Codex 以加载新提示（重新启动您的 CLI 会话， 如果您正在使用它，请重新加载 IDE 扩展）。

预期：在斜杠命令菜单中输入 `/prompts:draftpr` 将显示您的自定义命令以及来自前言的描述，并提示文件和 PR 标题是可选的。

## 添加元数据和参数

Codex 读取提示元数据，并在下次会话启动时解析占位符。

- **描述：**在弹出窗口中的命令名称下显示。在 YAML 前言中设置为 `description:`。
- **参数提示：**用 `argument-hint: KEY=<value>` 文档记录预期参数。
- **位置占位符：**`$1` 到 `$9` 从您在命令后提供的以空格分隔的参数中扩展。`$ARGUMENTS` 包含它们所有。
- **命名占位符：**使用大写名称，如 `$FILE` 或 `$TICKET_ID`，并将值作为 `KEY=value` 提供。带空格的值请加引号（例如，`FOCUS="loading state"`）。
- **字面美元符号：**写 `$$` 以在扩展提示中发出一个单一的 `$`。

编辑提示文件后，重新启动 Codex 或打开一个新聊天以加载更新。Codex 会忽略提示目录中的非 Markdown 文件。

## 调用和管理自定义命令

1. 在 Codex（CLI 或 IDE 扩展）中，输入 `/` 打开斜杠命令菜单。

2. 输入 `prompts:` 或提示名称，例如 `/prompts:draftpr`。

3. 提供所需参数：

```
/prompts:draftpr FILES="src/pages/index.astro src/lib/api.ts" PR_TITLE="添加英雄动画"
```

4. 按 Enter 发送扩展的指令（当您不需要某个参数时可以跳过）。

预期：Codex 扩展 `draftpr.md` 的内容，用您提供的参数替换占位符，然后将结果作为消息发送。

通过编辑或删除 `~/.codex/prompts/` 下的文件来管理提示。Codex 只扫描该文件夹中的顶级 Markdown 文件，因此将每个自定义提示直接放在 `~/.codex/prompts/` 下，而不是在子目录中。
