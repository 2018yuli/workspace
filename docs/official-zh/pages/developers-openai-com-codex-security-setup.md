复制页面

本页面指导您从初始访问到审查发现和 Codex 安全中的修复请求。

首先确认您已设置 Codex Cloud。如果没有，请参阅 [Codex\\
Cloud](https://developers.openai.com/codex/cloud) 开始。

## 1\. 访问和环境

Codex 安全扫描通过 [Codex Cloud](https://developers.openai.com/codex/cloud) 连接的 GitHub 仓库。

- 确认您的工作区可以访问 Codex 安全。
- 确认您要扫描的仓库在 Codex Cloud 中可用。

访问 [Codex 环境](https://chatgpt.com/codex/settings/environments) 并检查仓库是否已有环境。如果没有，请在继续之前创建一个。

[打开环境](https://chatgpt.com/codex/settings/environments)

![Codex 环境](https://developers.openai.com/_astro/create_environment.M-EPszPH.png)

## 2\. 新的安全扫描

环境存在后，访问 [创建安全扫描](https://chatgpt.com/codex/security/scans/new) 并选择您刚刚连接的仓库。

[创建安全扫描](https://chatgpt.com/codex/security/scans/new)

Codex 安全首先从最新提交向后扫描仓库。它利用这些信息在新的提交到达时构建和刷新扫描上下文。

要配置一个仓库：

1. 选择 GitHub 组织。
2. 选择仓库。
3. 选择您要扫描的分支。
4. 选择环境。
5. 选择一个 **历史时间窗口**。较长的窗口提供更多上下文，但回填所需时间较长。
6. 点击 **创建**。

![创建安全扫描](https://developers.openai.com/_astro/create_scan.mEjmf4U_.png)

## 3\. 初始扫描可能需要一些时间

当您创建扫描时，Codex 安全首先在所选的历史时间窗口内进行提交级别的安全检查。
初始回填可能需要几个小时，尤其是对于较大的仓库或较长的时间窗口。
如果发现内容没有立即显示，这是正常现象。在打开工单或故障排除之前，请等待初始扫描完成。

初始扫描设置自动且全面。这可能需要几个小时。如果第一次发现的结果延迟出现，请不要惊慌。

## 4\. 审查扫描并改善威胁模型

[审查扫描](https://chatgpt.com/codex/security/scans)

![Codex 安全中的威胁模型编辑器](https://developers.openai.com/_astro/review_threat_model.JTLMQEmx.png)

当初始扫描结束后，打开扫描并审查生成的威胁模型。
在初始发现出现后，更新威胁模型以匹配您的架构、信任边界和业务上下文。
这有助于 Codex 安全为您的团队排名问题。

如果您希望扫描结果发生变化，您可以使用更新后的范围、优先事项和假设编辑威胁模型。

在初始发现出现后，重新查看模型，以确保扫描指导与当前优先事项保持一致。
保持最新有助于 Codex 安全提出更好的建议。

有关威胁模型及其如何影响重要性和分类的更深层次解释，请参阅 [改进威胁模型](https://developers.openai.com/codex/security/threat-model)。

## 5\. 审查发现并修补

在初始回填完成后，从 **发现** 视图中审查发现内容。

[打开发现](https://chatgpt.com/codex/security/findings)

您可以使用两个视图：

- **推荐发现**：一个不断更新的仓库中最关键问题的前 10 名列表
- **所有发现**：一个可排序、可过滤的发现表，涵盖整个仓库

![推荐发现视图](https://developers.openai.com/_astro/aardvark_recommended_findings.Dj6eIPV0_ZhRjOL.webp?dpl=dpl_7VPtn8exRw6M89oqTCrgyMZz2kmj)

单击一个发现以打开其详细页面，其中包括：

- 问题的简明描述
- 关键元数据，例如提交详情和文件路径
- 关于影响的上下文推理
- 相关代码摘录
- 可用时的调用路径或数据流上下文
- 验证步骤和验证输出

您可以审查每个发现并直接从发现详细页面创建 PR。

[审查发现并创建 PR](https://chatgpt.com/codex/security/findings)
