Codex 在您将其视为一个具有明确上下文和“完成”清晰定义的队友时效果最佳。
此页面提供了 Codex IDE 扩展、Codex CLI 和 Codex 云的端到端工作流示例。

## 如何阅读这些示例

每个工作流包括：

- **何时使用**以及哪个 Codex 表面最合适（IDE、CLI 或云）。
- **步骤**，带有示例用户提示。
- **上下文说明**：Codex 自动看到的内容与您需要附加的内容。
- **验证**：如何检查输出。

> **注意：** IDE 扩展会自动将您打开的文件作为上下文包含在内。在 CLI 中，您通常需要明确提及路径（或使用 `/mention` 和 `@` 路径自动补全附加文件）。

## 解释代码库

在您入职、继承服务或试图理解协议、数据模型或请求流程时使用此功能。

### IDE 扩展工作流（适合本地探索）

1. 打开最相关的文件。

2. 选择您关心的代码（可选，但推荐）。

3. 提示 Codex：

```
解释请求如何通过所选代码流动。

包括：
- 每个模块的职责简短摘要
- 验证的数据及其位置
- 更改时需要注意的一两个“陷阱”
```

验证：

- 请求一个您可以快速验证的图表或检查清单：

```
将请求流程总结为编号步骤列表。然后列出涉及的文件。
```

### CLI 工作流（在您希望获得转录 + shell 命令时）

1. 开始一个交互式会话：

```
codex
```

2. 附加文件（可选）并提示：

```
我需要理解此服务使用的协议。阅读 @foo.ts @schema.ts，并解释模式和请求/响应流。关注必需与可选字段以及向后兼容规则。
```

上下文说明：

- 您可以在 composer 中使用 `@` 插入来自工作区的文件路径，或者 `/mention` 附加特定文件。

## 修复 bug

在您有可以本地复现的失败行为时使用此功能。

### CLI 工作流（紧密循环与重现和验证）

1. 在 repo 根目录启动 Codex：

```
codex
```

2. 给 Codex 一个重现食谱，以及您怀疑的文件：

```
Bug: 点击设置屏幕上的“保存”有时显示“已保存”，但不持久化更改。

重现：
1）启动应用程序：npm run dev
2）转到 /settings
3）切换“启用警报”
4）点击保存
5）刷新页面：切换恢复

约束：
- 不改变 API 形状。
- 保持修复最小，并在可行的情况下添加回归测试。

首先在本地重复 bug，然后提议修补程序并运行检查。
```

上下文说明：

- 由您提供：重现步骤和约束（这些比高层描述更重要）。
- 由 Codex 提供：命令输出、发现的调用位置以及触发的任何堆栈跟踪。

验证：

- Codex 应在修复后重新运行重现步骤。
- 如果您有标准检查管道，请要求其运行：

```
修复后，运行 lint + 最小相关的测试套件。报告命令和结果。
```

### IDE 扩展工作流

1. 打开您认为 bug 所在的文件，以及其最近的调用者。

2. 提示 Codex：

```
找到导致“已保存”而不持久化更改的 bug。在提议修复后，告诉我如何在 UI 中验证。
```

## 编写测试

在您想要非常明确地测试的范围时使用此功能。

### IDE 扩展工作流（基于选择）

1. 打开包含函数的文件。

2. 选择定义函数的行。从命令面板选择“添加到 Codex 线程”以将这些行添加到上下文中。

3. 提示 Codex：

```
为此函数编写单元测试。按照其他测试中使用的约定。
```

上下文说明：

- 由“添加到 Codex 线程”命令提供：所选行（这是“行号”范围），以及打开的文件。

### CLI 工作流（路径 + 提示中描述的行范围）

1. 启动 Codex：

```
codex
```

2. 提示函数名称：

```
为 @transform.ts 中的 invert_list 函数添加测试。涵盖快乐路径和边缘情况。
```

## 从截图原型

在您有设计模型、截图或 UI 参考时使用此功能，并希望快速获得工作原型。

### CLI 工作流（图像 + 提示）

1. 在本地保存屏幕截图（例如 `./specs/ui.png`）。

2. 运行 Codex：

```
codex
```

3. 将图像文件拖入终端以将其附加到提示中。

4. 随后使用约束和结构：

```
基于此图像创建新的仪表板。

约束：
- 使用 react、vite 和 tailwind。代码使用 typescript 编写。
- 尽可能匹配间距、排版和布局。

交付物：
- 一个新的路由/页面，用于呈现 UI
- 所需的任何小组件
- 带有本地运行说明的 README.md
```

上下文说明：

- 图像提供视觉要求，但您仍需指定实现约束（框架、路由、组件样式）。
- 为了获得最佳效果，请在文本中包含任何不明显的行为（悬停状态、验证规则、键盘交互）。

验证：

- 要求 Codex 运行开发服务器（如果允许），并告诉您确切的查看原型的本地 URL/路由：

```
启动开发服务器并告诉我本地 URL/路由以查看原型。
```

### IDE 扩展工作流（图像 + 现有文件）

1. 在 Codex 聊天中附加图像（拖放或粘贴）。

2. 提示 Codex：

```
创建一个新的设置页面。使用附加的截图作为目标 UI。
遵循该项目中其他文件的设计和视觉模式。
```

## 通过实时更新迭代 UI

在您希望 Codex 编辑代码时进行紧密的“设计 → 调整 → 刷新 → 调整”循环时使用此功能。

### CLI 工作流（运行 Vite，然后通过小提示迭代）

1. 启动 Codex：

```
codex
```

2. 在另一个终端窗口中启动开发服务器：

```
npm run dev
```

3. 提示 Codex 进行更改：

```
为着陆页提出 2-3 个样式改进。
```

4. 选择一个方向，并通过小的、具体的提示迭代：

```
选择选项 2。

只更改标题：
- 使排版更具编辑性
- 增加空白
- 确保在移动端仍然看起来不错
```

5. 在有针对性的请求中重复：

```
下一次迭代：减少视觉噪声。
保持布局，但简化颜色并去除冗余边框。
```

验证：

- 在浏览器中“实时”查看更改，因为代码正在更新。
- 提交您喜欢的更改，并还原您不喜欢的更改。
- 如果您还原或修改更改，请告知 Codex，以便它在下一个提示时不会覆盖更改。

## 将重构委派给云

在您希望仔细设计（本地上下文、快速检查），然后将长时间实现外包给可以并行运行的云任务时使用此功能。

### 本地规划（IDE）

1. 确保您的当前工作已提交或至少已暂存，以便您可以干净地比较更改。

2. 请求 Codex 生成重构计划。如果您具有可用的 `$plan` 技能，请显式调用它：

```
$plan

我们需要重构 auth 子系统以：
- 拆分责任（令牌解析与会话加载与权限）
- 减少循环导入
- 提高可测试性

约束：
- 不可见行为变化
- 保持公共 API 稳定
- 包含逐步迁移计划
```

3. 审查计划并谈判更改：

```
将计划修改为：
- 指定每个里程碑中移动的确切文件
- 包括回滚策略
```

上下文说明：

- 规划在 Codex 可以在本地扫描当前代码时效果最佳（入口点、模块边界、依赖图线索）。

### 云委派（IDE → 云）

1. 如果尚未完成，请设置一个 [Codex 云环境](https://developers.openai.com/codex/cloud/environments)。

2. 在提示 composer 下方单击云图标并选择您的云环境。

3. 当您输入下一个提示时，Codex 会创建一个新的线程到云中，该线程会传递现有的线程上下文（包括计划和任何本地源更改）。

```
实施计划中的第 1 里程碑。
```

4. 审查云差异，如果需要，进行迭代。

5. 直接从云创建 PR 或将更改拉取到本地以进行测试和完成。

6. 在计划的额外里程碑上进行迭代。

## 进行本地代码审查

在您希望在提交或创建 PR 之前获得第二双眼睛时使用此功能。

### CLI 工作流（审查您的工作树）

1. 启动 Codex：

```
codex
```

2. 运行审查命令：

```
/review
```

3. 可选：提供自定义关注指令：

```
/review 关注边缘情况和安全问题
```

验证：

- 根据审查反馈应用修复，然后重新运行 `/review` 以确认问题已解决。

## 审查 GitHub 拉取请求

在您希望获得审查反馈而不将分支拉取到本地时使用此功能。

在使用此功能之前，请在您的存储库中启用 Codex **代码审查**。请参阅 [代码审查](https://developers.openai.com/codex/integrations/github)。

### GitHub 工作流（基于评论）

1. 在 GitHub 上打开拉取请求。

2. 留下评论以标签 Codex 并明确关注区域：

```
@codex review
```

3. 可选：提供更明确的指令。

```
@codex review 安全漏洞和安全问题
```

## 更新文档

在您需要进行准确清晰的文档更改时使用此功能。

### IDE 或 CLI 工作流（本地编辑 + 本地验证）

1. 确定要更改的文档文件并打开它们（IDE）或 `@` 提及它们（IDE 或 CLI）。

2. 提示 Codex 说明范围和验证要求：

```
更新“高级功能”文档以提供身份验证故障排除指导。验证所有链接有效。
```

3. 在 Codex 草拟更改后，查看文档并根据需要进行迭代。

验证：

- 阅读渲染的页面。
