## 下载并更新 Codex 应用

从[Microsoft Store](https://get.microsoft.com/installer/download/9PLM9XGG6VKS?cid=website_cta_psi)下载 Codex 应用。

然后按照[快速开始](https://developers.openai.com/codex/quickstart?setup=app)进行操作。

要更新应用，请打开 Microsoft Store，转到 **下载**，然后点击 **检查更新**。之后，Store 会安装最新版本。

对于企业，管理员可以通过企业管理工具使用 Microsoft Store 应用分发来部署应用。

如果您更喜欢命令行安装路径，或者需要替代打开 Microsoft Store UI 的方法，请运行：

```
winget install Codex -s msstore
```

## 原生沙盒

Windows 上的 Codex 应用支持当代理在 PowerShell 中运行时的原生 [Windows 沙盒](https://developers.openai.com/codex/windows#windows-sandbox)，并且在您运行代理时使用 [Windows 子系统 Linux 2 (WSL2)](https://developers.openai.com/codex/app/windows#windows-subsystem-for-linux-wsl) 时使用 Linux 沙盒。当您在任何模式下应用沙盒保护时，请在 Composer 中将沙盒权限设置为 **默认权限**，然后再向 Codex 发送消息。

以完全访问模式运行 Codex 意味着 Codex 不受限于您的项目目录，并且可能会执行导致数据丢失的意外破坏性操作。保持沙盒边界，并使用 [规则](https://developers.openai.com/codex/rules) 进行有针对性的例外情况，或将您的 [批准策略设置为\\
从不](https://developers.openai.com/codex/agent-approvals-security#run-without-approval-prompts)，以使 Codex 尝试解决问题而不请求更高权限，基于您的 [批准和安全设置](https://developers.openai.com/codex/agent-approvals-security)。

## Windows 子系统 Linux (WSL)

默认情况下，Codex 应用使用 Windows 原生代理。这意味着代理在 PowerShell 中运行命令。该应用仍然可以通过在需要时使用 `wsl` CLI 来处理位于 Windows 子系统 Linux 2 (WSL2) 的项目。

如果您想从 WSL 文件系统添加项目，请点击 **添加新项目** 或者按 `Ctrl` + `O`，然后在文件资源管理器窗口中输入 `\\wsl$\`。从那里，选择您的 Linux 发行版和您想要打开的文件夹。

如果您计划继续使用 Windows 原生代理，建议将项目存储在 Windows 文件系统中，并通过 `/mnt/<drive>/...` 从 WSL 访问它们。此设置比直接从 WSL 文件系统打开项目更可靠。

如果您希望代理本身在 WSL2 中运行，请打开 **[设置](codex://settings)**，将代理从 Windows 原生切换到 WSL，并 **重启应用**。更改在重启之前不会生效。您的项目应该在重启后保持不变。

WSL1 的支持在 Codex `0.114` 版本中。自 Codex `0.115` 开始，Linux 沙盒迁移到 `bubblewrap`，因此不再支持 WSL1。

您可以独立于代理配置集成终端。有关终端选项，请参见[根据您的开发设置进行定制](https://developers.openai.com/codex/app/windows#customize-for-your-dev-setup)。您可以将代理保留在 WSL 中，同时在终端中使用 PowerShell，或根据您的工作流为二者使用 WSL。

## 有用的开发工具

Codex 在一些常用的开发工具已安装时效果最佳：

- **Git**：使 Codex 应用中的审查面板提供动力，并允许您检查或还原更改。
- **Node.js**：代理用于更高效地执行任务的常见工具。
- **Python**：代理用于更高效地执行任务的常见工具。
- **.NET SDK**：在您想要构建原生 Windows 应用时非常有用。
- **GitHub CLI**：使 Codex 应用中的 GitHub 特定功能提供动力。

通过将以下内容粘贴到[集成终端](https://developers.openai.com/codex/app/features#integrated-terminal)中或请求 Codex 安装它们，使用默认 Windows 包管理器 `winget` 安装它们：

```
winget install --id Git.Git
winget install --id OpenJS.NodeJS.LTS
winget install --id Python.Python.3.14
winget install --id Microsoft.DotNet.SDK.10
winget install --id GitHub.cli
```

安装 GitHub CLI 后，运行 `gh auth login` 来启用应用中的 GitHub 功能。

如果您需要不同的 Python 或 .NET 版本，请将包 ID 更改为您想要的版本。

## 故障排除和常见问题

### 以提升权限运行命令

如果您需要 Codex 以提升权限运行命令，请以管理员身份启动 Codex 应用本身。安装后，打开开始菜单，找到 Codex，并选择以管理员身份运行。Codex 代理继承该权限级别。

### PowerShell 执行策略阻止命令

如果您从未在 PowerShell 中使用过 Node.js 或 `npm` 等工具，Codex 代理或集成终端可能会出现执行策略错误。

如果 Codex 为您创建 PowerShell 脚本，也可能发生这种情况。在这种情况下，您可能需要一个较不严格的执行策略，才能运行它们。

错误可能类似于以下内容：

```
npm.ps1 无法加载，因为在此系统上禁用了运行脚本。
```

常见的修复方法是将执行策略设置为 `RemoteSigned`：

```
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned
```

有关详细信息和其他选项，请在更改策略之前查看 Microsoft 的 [执行策略指南](https://learn.microsoft.com/en-us/powershell/module/microsoft.powershell.core/about/about_execution_policies)。

### Windows 上的本地环境脚本

如果您的[本地环境](https://developers.openai.com/codex/app/local-environments)使用跨平台命令，如 `npm` 脚本，则您可以保持一个共享的设置脚本或每个平台的一组操作。

如果您需要 Windows 特定行为，请创建 Windows 特定的设置脚本或 Windows 特定操作。

操作在您集成终端使用的环境中运行。请参见[根据您的开发设置进行定制](https://developers.openai.com/codex/app/windows#customize-for-your-dev-setup)。

本地设置脚本在代理环境中运行：如果代理使用 WSL，则在 WSL 中运行，否则在 PowerShell 中运行。

### 与 WSL 共享配置、身份验证和会话

Windows 应用使用与 Windows 上原生 Codex 相同的 Codex 主目录：`%USERPROFILE%\.codex`。

如果您还在 WSL 中运行 Codex CLI，则 CLI 默认使用 Linux 主目录，因此不会自动与 Windows 应用共享配置、缓存的身份验证或会话历史。

要共享它们，请使用以下方法之一：

- 将 WSL `~/.codex` 与您的文件系统上的 `%USERPROFILE%\.codex` 同步。
- 通过设置 `CODEX_HOME` 将 WSL 指向 Windows Codex 主目录：

```
export CODEX_HOME=/mnt/c/Users/<windows-user>/.codex
```

如果您希望在每个 shell 中都有该设置，请将其添加到您的 WSL shell 配置文件中，例如 `~/.bashrc` 或 `~/.zshrc`。

### Git 功能不可用

如果您在 Windows 上未原生安装 Git，则该应用无法使用某些功能。通过 PowerShell 或 `cmd.exe` 安装它：`winget install Git.Git`。

### 从 `\\wsl$` 打开的项目未检测到 Git

目前，如果您想用 Windows 原生代理处理也可以从 WSL 访问的项目，最可靠的解决方法是将项目存储在本地 Windows 驱动器上，并通过 `/mnt/<drive>/...` 在 WSL 中访问它。

### `Cmder` 没有列在打开对话框中

如果已安装 `Cmder` 但在 Codex 的打开对话框中未显示它，请将其添加到 Windows 开始菜单：右键单击 `Cmder` 并选择 **添加到开始**，然后重启 Codex 或重新启动计算机。
