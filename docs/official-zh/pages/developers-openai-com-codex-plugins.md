## 概述

插件将技能、应用集成和 MCP 服务器捆绑成可重用的工作流，以供 Codex 使用。

扩展 Codex 的功能，例如：

- 安装 Gmail 插件使 Codex 能够读取和管理 Gmail。
- 安装 Google Drive 插件以在 Drive、Docs、Sheets 和 Slides 之间工作。
- 安装 Slack 插件以总结频道或草拟回复。

插件可以包含：

- **技能：** 特定类型工作的可重用指令。Codex 可以在需要时加载它们，以遵循正确的步骤并使用正确的参考或辅助脚本来完成任务。
- **应用：** 连接到 GitHub、Slack 或 Google Drive 等工具，因此 Codex 能够从这些工具读取信息并在其中采取行动。
- **MCP 服务器：** 提供 Codex 访问更多工具或共享信息的服务，通常来自您本地项目外部的系统。

更多插件功能即将推出。

## 使用和安装插件

### Codex 应用中的插件目录

在 Codex 应用中打开 **插件** 以浏览和安装经过策划的插件。

### CLI 中的插件目录

在 Codex CLI 中，运行以下命令以打开插件列表：

```
codex
/plugins
```

![Codex CLI 中的插件列表](https://developers.openai.com/images/codex/plugins/cli_light.png)![Codex CLI 中的插件列表](https://developers.openai.com/images/codex/plugins/codex-plugin-cli.png)

![Codex CLI 中的插件列表](https://developers.openai.com/images/codex/plugins/cli_light.png)![Codex CLI 中的插件列表](https://developers.openai.com/images/codex/plugins/codex-plugin-cli.png)

CLI 插件浏览器按市场对插件进行分组。使用市场选项卡切换源，打开插件以检查详细信息，并按 `Space` 切换已安装插件的启用状态。

### 安装和使用插件

打开插件目录后：

1. 搜索或浏览插件，然后打开其详细信息。
2. 选择安装按钮。在应用中，选择加号按钮或 **添加到 Codex**。在 CLI 中，选择 `安装插件`。
3. 如果插件需要外部应用，请在提示时连接。如果一些插件在安装过程中要求您进行身份验证，其他插件会等待您第一次使用它们时再进行。
4. 安装后，启动新线程并要求 Codex 使用该插件。

安装插件后，您可以直接在提示窗口中使用它：

![Codex 插件页面](https://developers.openai.com/images/codex/plugins/plugin-github-invoke.png)![Codex 插件页面](https://developers.openai.com/images/codex/plugins/plugin-github-invoke-dark.png)

![Codex 插件页面](https://developers.openai.com/images/codex/plugins/plugin-github-invoke.png)![Codex 插件页面](https://developers.openai.com/images/codex/plugins/plugin-github-invoke-dark.png)

直接描述任务

询问您想要的结果，例如“总结今天未读的 Gmail 线程”或“从 Google Drive 中提取最新的发布说明”。

在您希望 Codex 为任务选择合适的已安装工具时使用此功能。

选择特定插件

键入 `@` 以显式调用插件或其捆绑的技能。

在您希望明确指出 Codex 应该使用哪个插件或技能时使用此功能。请参见 [Codex 应用命令](https://developers.openai.com/codex/app/commands) 和 [技能](https://developers.openai.com/codex/skills)。

### 权限和数据共享工作的方式

安装插件使其工作流可用于 Codex，但您现有的 [批准设置](https://developers.openai.com/codex/agent-approvals-security) 仍然适用。任何连接的外部服务仍受其自身身份验证、隐私和数据共享政策的约束。

- 捆绑的技能在您安装插件后立即可用。
- 如果插件包含应用，Codex 可能会在设置期间或第一次使用时提示您在 ChatGPT 中安装或登录这些应用。
- 如果插件包含 MCP 服务器，它们可能需要额外的设置或身份验证才能使用。
- 当 Codex 通过捆绑的应用发送数据时，应用的条款和隐私政策适用。

### 删除或关闭插件

要删除插件，请从插件浏览器重新打开它并选择 **卸载插件**。

卸载插件将从 Codex 中移除插件包，但捆绑的应用直到您在 ChatGPT 中管理它们时仍然保持安装状态。

如果您希望保留已安装的插件但关闭它，请将其条目设置为 `~/.codex/config.toml` 中的 `enabled = false`，然后重启 Codex：

```
[plugins."gmail@openai-curated"]
enabled = false
```
