Codex 应用是一个专注的桌面体验，旨在并行处理 Codex 线程，具有内置的工作树支持、自动化和 Git 功能。

大多数 Codex 应用功能在 macOS 和 Windows 上均可用。以下部分注意平台特定的例外情况。

* * *

## 在项目间多任务处理

使用一个 Codex 应用窗口在项目间运行任务。为每个代码库添加一个项目，并在需要时切换它们。

如果您使用过 [Codex CLI](https://developers.openai.com/codex/cli)，项目就像是在特定目录中启动一个会话。

如果您在一个单一的代码库中有两个或多个应用或软件包，请将不同的项目分成单独的应用项目，这样 [沙盒](https://developers.openai.com/codex/agent-approvals-security) 中仅包括该项目的文件。

## 模式

每个线程在选择的模式下运行。开始线程时，您可以选择：

- **本地**：直接在您当前的项目目录中工作。
- **工作树**：在 Git 工作树中隔离更改。[了解更多](https://developers.openai.com/codex/app/worktrees)。
- **云**：在配置的云环境中远程运行。

**本地**和**工作树**线程将在您的计算机上运行。

要获取完整的术语表和概念，请浏览 [概念部分](https://developers.openai.com/codex/prompting)。

## 工作树支持

创建新线程时，选择**本地**或**工作树**。**本地**直接在您的项目中工作。**工作树**创建一个新的 [Git 工作树](https://git-scm.com/docs/git-worktree)，因此更改会与您常规项目隔离。

当您想尝试一个新想法而不触及当前工作，或者希望 Codex 并行运行独立任务在同一项目中时，请使用**工作树**。

自动化在 Git 仓库的专用后台工作树中运行，并在非版本控制项目的项目目录中直接运行。

[了解有关在 Codex 应用中使用工作树的更多信息。](https://developers.openai.com/codex/app/worktrees)

## 集成终端

每个线程包括一个内置终端，范围限于当前项目或工作树。通过应用右上角的终端图标切换，或按 `Cmd` + `J`。

使用终端验证更改、运行脚本和执行 Git 操作，而无需离开应用。Codex 还可以读取当前终端输出，因此它可以检查正在运行的开发服务器的状态或在与您合作时参考失败的构建。

常见任务包括：

- `git status`
- `git pull --rebase`
- `pnpm test` 或 `npm test`
- `pnpm run lint` 或类似项目命令

如果您定期运行任务，可以在您的 [本地环境](https://developers.openai.com/codex/app/local-environments) 中定义一个 **操作**，为您的 Codex 应用窗口顶部添加一个快捷按钮。

请注意，`Cmd` + `K` 在 Codex 应用中打开命令面板。它不会清除终端。要清除终端，请使用 `Ctrl` + `L`。

## 应用内浏览器

使用 [应用内浏览器](https://developers.openai.com/codex/app/browser) 预览、审核和评论本地开发服务器、文件支持的预览和不需要登录的公共页面，同时您迭代网页应用。

应用内浏览器不支持身份验证流程、已登录页面、您的常规浏览器配置文件、Cookies、扩展或现有标签。

使用浏览器评论标记页面上的特定元素或区域，然后要求 Codex 处理该反馈。

当您希望 Codex 直接操作页面时，请使用 [浏览器使用](https://developers.openai.com/codex/app/browser#browser-use) 用于本地开发服务器和文件支持的页面。您可以从设置中管理浏览器插件、允许的网站和阻止的网站。

## 处理非代码工件

当任务产生非代码工件时，侧边栏可以预览 PDF 文件、电子表格、文档和演示文稿。将源数据、预期文件类型、结构和您关注的审查标准提供给 Codex。

对于电子表格和演示文稿，描述重要的表格、列、图表、幻灯片部分和检查内容。要求 Codex 解释它保存了什么输出以及它如何检查结果。

使用任务侧边栏查看 Codex 在线程运行时正在做什么。它可以展示代理的计划、来源、生成的工件和任务摘要，以便您可以引导工作、检查生成的文件，并决定需要重新处理什么。

## 与 IDE 扩展同步

如果您在编辑器中安装了 [Codex IDE 扩展](https://developers.openai.com/codex/ide)，您的 Codex 应用和 IDE 扩展会在同一项目中自动同步。

同步时，您将在 Codex 应用作曲器中看到 **IDE 上下文** 选项。启用“自动上下文”时，Codex 应用会跟踪您正在查看的文件，因此您可以间接引用它们（例如，“这个文件是关于什么的？”）。您还可以在 Codex 应用内部看到正在运行的线程，在 IDE 扩展中也是如此。

如果您不确定该应用是否包含上下文，可以切换关闭并再次提出相同的问题以比较结果。

## 线程自动化

自动化也可以附加到单个线程。这些线程自动化是定期的唤醒呼叫，保留线程的上下文，以便 Codex 可以检查长期运行的工作、轮询源以获取新信息，或继续后续循环。将它们用于应保持按计划返回到同一对话的心跳式自动化。

当下一次运行依赖于当前对话时，使用线程自动化。当您希望 Codex 为一个或多个项目启动新的定期任务时，使用独立或项目 [自动化](https://developers.openai.com/codex/app/automations)。

## 批准和沙盒化

您的批准和沙盒设置限制 Codex 的操作。

- 批准决定 Codex 在运行命令之前何时暂停以获取权限。
- 沙盒控制 Codex 可以使用的目录和网络访问。

当您看到提示“只批准一次”或“批准此次会话”时，您授予工具执行的不同权限范围。如果您不确定，请批准最狭窄的选项并继续迭代。

默认情况下，Codex 的作用域工作在当前项目中。在大多数情况下，这就是正确的限制。

如果您的任务需要跨多个仓库或目录进行操作，建议开启单独的项目或使用工作树，而不是要求 Codex 在项目根目录外活动。

如果 [自动审核](https://developers.openai.com/codex/agent-approvals-security#automatic-approval-reviews) 在您的工作区可用，您可以从权限选择器中选择它。它保持相同的沙盒边界，但通过配置的审核政策路由合格的批准请求，而不是等待您。

有关总体概述，请参见 [沙盒化](https://developers.openai.com/codex/concepts/sandboxing)。有关配置详细信息，请参阅 [代理批准与安全文档](https://developers.openai.com/codex/agent-approvals-security)。

## MCP 支持

Codex 应用、CLI 和 IDE 扩展共享 [模型上下文协议 (MCP)](https://developers.openai.com/codex/mcp) 设置。如果您已经在其中一个中配置了 MCP 服务器，它们会被其他自动采用。要配置新服务器，请在应用的设置中打开 MCP 部分，启用推荐的服务器或将新服务器添加到您的配置中。

## 网络搜索

Codex 配备了第一个方网络搜索工具。对于 Codex 应用中的本地任务，Codex 默认启用网络搜索，并提供来自网络搜索缓存的结果。如果您将沙盒配置为 [完全访问](https://developers.openai.com/codex/agent-approvals-security)，网络搜索默认为实时结果。请参见 [配置基础](https://developers.openai.com/codex/config-basic)以禁用网络搜索或切换到获取最新数据的实时结果。

## 图像生成

要求 Codex 在线程中直接生成或编辑图像。这对 UI 资产、横幅、背景、插图、精灵表和您希望与代码一起创建的占位符非常有用。当您希望 Codex 转换或扩展现有资产时，请添加参考图像。

您可以自然语言提问，或通过在提示中包含 `$imagegen` 明确调用图像生成技能。

内置图像生成使用 `gpt-image-2`，计入您一般的 Codex 使用限额，并且平均速度比不使用图像生成的类似轮次快 3-5 倍，具体取决于图像质量和大小。有关详细信息，请参见 [定价](https://developers.openai.com/codex/pricing#image-generation-usage-limits)。有关提示技巧和模型详细信息，请参见 [图像生成指南](https://developers.openai.com/api/docs/guides/image-generation)。

对于较大批量的图像生成，在您的环境变量中设置 `OPENAI_API_KEY`，并要求 Codex 通过 API 生成图像，以便应用 API 定价。

## 图像输入

您可以将图像拖放到提示作曲器中以将其作为上下文包含。按住 `Shift` 同时放入图像以将图像添加到上下文中。

您还可以要求 Codex 查看您系统上的图像。通过向 Codex 提供工具来截取您正在处理的应用的屏幕截图，Codex 可以验证它正在进行的工作。

## 聊天

聊天是您可以在任务不需要特定项目文件夹或 Git 仓库时开始的线程。将它们用于研究、分流、规划、插件密集型工作流程和其他对话，其中 Codex 应该使用连接的工具而不是编辑代码库。

聊天使用 Codex 管理的 `threads` 目录作为其工作位置。默认位置是 `~/.codex/threads`。

## 记忆

[记忆](https://developers.openai.com/codex/memories)，在可用的情况下，让 Codex 将以往任务中的有用上下文带入未来的线程。它们对于稳定的偏好、项目约定、重复的工作模式和已知的陷阱最为有用，否则它们将需要重复。

## 通知

默认情况下，当任务完成或在应用程序后台时需要批准时，Codex 应用会发送通知。

在 Codex 应用设置中，您可以选择从不发送通知或始终发送通知，即使在应用程序处于焦点时。

## 保持计算机唤醒

由于您的任务可能需要一些时间才能完成，您可以通过在应用设置中启用“运行时防止睡眠”切换，来让 Codex 应用防止计算机进入睡眠状态。
