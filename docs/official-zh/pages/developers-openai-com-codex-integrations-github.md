使用 Codex 审查拉取请求，而无需离开 GitHub。添加一个拉取请求评论，输入 `@codex review`，Codex 会回复一个标准的 GitHub 代码审查。

## 设置代码审查

1. 设置 [Codex 云](https://developers.openai.com/codex/cloud)。
2. 转到 [Codex 设置](https://chatgpt.com/codex/settings/code-review)，为您的存储库启用 **代码审查**。

![Codex 设置显示代码审查切换](https://developers.openai.com/images/codex/code-review/code-review-settings.png)

## 请求审查

1. 在拉取请求评论中，提及 `@codex review`。
2. 等待 Codex 响应（👀）并发布审查。

![含 @codex review 的拉取请求评论](https://developers.openai.com/images/codex/code-review/review-trigger.png)

Codex 在拉取请求上发布审查，就像队友一样。

![在拉取请求上的 Codex 代码审查示例](https://developers.openai.com/images/codex/code-review/review-example.png)

## 启用自动审查

如果您希望 Codex 自动审查每个拉取请求，请在 [Codex 设置](https://chatgpt.com/codex/settings/code-review)中启用 **自动审查**。Codex 每当有新的 PR 提交审查时，都会发布一个审查，而不需要 `@codex review` 评论。

## 自定义 Codex 审查内容

Codex 在您的存储库中搜索 `AGENTS.md` 文件，并遵循您包含的任何 **审查指南**。

要为存储库设置指南，请添加或更新一个顶层的 `AGENTS.md`，包括如下部分：

```
## 审查指南

- 不要记录个人身份信息。
- 确保身份验证中间件包装每个路由。
```

Codex 应用来自最近 `AGENTS.md` 的指导到每个更改的文件。如果特定软件包需要额外审核，您可以将更具体的说明放在树中的更深层次。

对于一次性的关注，可以在您的拉取请求评论中添加，例如：

`@codex review for security regressions`

在 GitHub 中，Codex 仅标记 P0 和 P1 问题。如果您希望 Codex 标记文档中的错别字，请在 `AGENTS.md` 中添加指导（例如，“将文档中的错别字视为 P1。”）。

## 给 Codex 其他任务

如果您在评论中提及 `@codex` 并包含其他内容而非 `review`，Codex 将使用您的拉取请求作为上下文开始一个 [云任务](https://developers.openai.com/codex/cloud)。

```
@codex 修复 CI 失败
