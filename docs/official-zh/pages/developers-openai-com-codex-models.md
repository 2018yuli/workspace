当你通过 ChatGPT 登录时，Codex 最适合使用上述模型。

你还可以将 Codex 指向任何支持 [聊天完成](https://platform.openai.com/docs/api-reference/chat) 或 [响应 API](https://platform.openai.com/docs/api-reference/responses) 的模型和提供者，以适应你的特定用例。

## 配置模型

### 配置你的默认本地模型

Codex CLI 和 IDE 扩展使用相同的 `config.toml` [配置文件](https://developers.openai.com/codex/config-basic)。要指定模型，请在配置文件中添加 `model` 条目。如果你不指定模型，Codex 应用、CLI 或 IDE 扩展将默认使用推荐模型。

```
model = "gpt-5.5"
```

如果 `gpt-5.5` 尚未在你的账户中可用，请使用 `gpt-5.4`。

### 临时选择不同的本地模型

在 Codex CLI 中，你可以在活动线程中使用 `/model` 命令更改模型。在 IDE 扩展中，你可以使用输入框下方的模型选择器选择模型。

要以特定模型启动新的 Codex CLI 线程或为 `codex exec` 指定模型，你可以使用 `--model`/`-m` 标志：

```
codex -m gpt-5.5
```

### 为云任务选择你的模型

目前，你无法更改 Codex 云任务的默认模型。
