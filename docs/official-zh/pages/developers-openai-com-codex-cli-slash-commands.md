斜杠命令让您快速、优先使用键盘控制 Codex。在创作工具中输入 `/` 打开斜杠弹出窗口，选择一个命令，Codex 将执行如切换模型、调整权限或总结长对话等操作，而不离开终端。

本指南展示了您如何：

- 找到适合任务的内置斜杠命令
- 使用 `/model`、`/fast`、`/personality`、`/permissions`、`/agent` 和 `/status` 等命令引导活动会话

## 内置斜杠命令

Codex 配备以下命令。打开斜杠弹出窗口并开始输入命令名称以过滤列表。

当任务已经在运行时，您可以输入斜杠命令并按 `Tab` 将其排队为下一个回合。Codex 在运行时解析排队的斜杠命令，因此命令菜单和错误会在当前回合完成后出现。斜杠补全在您排队命令之前仍然有效。

`/quit` 和 `/exit` 都退出 CLI。仅在保存或提交任何重要工作后使用它们。

`/approvals` 命令仍然作为别名工作，但不再出现在斜杠弹出列表中。

## 使用斜杠命令控制您的会话

以下工作流程使您的会话保持在轨道上而无需重启 Codex。

### 使用 `/model` 设置活动模型

1. 启动 Codex 并打开创作工具。
2. 输入 `/model` 然后按回车。
3. 从弹出窗口中选择一个模型，例如 `gpt-4.1-mini` 或 `gpt-4.1`。

预期：Codex 在转录文本中确认新模型。运行 `/status` 确认更改。

### 使用 `/fast` 切换快速模式

1. 输入 `/fast on`、`/fast off` 或 `/fast status`。
2. 如果您希望设置保持有效，在 Codex 提供保存时确认更新。

预期：Codex 报告当前线程的快速模式是打开还是关闭。在 TUI 页脚中，您还可以使用 `/statusline` 显示快速模式状态行项目。

### 使用 `/personality` 设置沟通风格

使用 `/personality` 修改 Codex 的沟通方式而无需重写提示。

1. 在活动对话中，输入 `/personality` 然后按回车。
2. 从弹出窗口中选择一种风格。

预期：Codex 在转录文本中确认新风格，并在后续响应中使用它。

Codex 支持 `friendly`、`pragmatic` 和 `none` 个性。使用 `none` 来禁用个性指令。

如果活动模型不支持个性特定的指令，Codex 会隐藏此命令。

### 使用 `/plan` 切换到计划模式

1. 输入 `/plan` 然后按回车以切换活动对话到计划模式。
2. 可选：提供内联提示文本（例如，`/plan 提出此服务的迁移计划`）。
3. 在使用内联 `/plan` 参数时，您可以粘贴内容或附加图像。

预期：Codex 进入计划模式并使用您的可选内联提示作为第一个计划请求。

当任务已经在运行时，`/plan` 暂时不可用。

### 使用 `/experimental` 切换实验性功能

1. 输入 `/experimental` 然后按回车。
2. 切换您想要的功能（例如，Apps 或 Smart Approvals），然后如果提示您，请重启 Codex。

预期：Codex 将您的功能选择保存到配置并在重启时应用。

### 使用 `/clear` 清除终端并开始新的聊天

1. 输入 `/clear` 然后按回车。

预期：Codex 清除终端，重置可见转录，并在同一 CLI 会话中开始一个新的聊天。

与 `Ctrl` + `L` 不同，`/clear` 开始新的对话。

`Ctrl` + `L` 仅会清除终端视图并保持当前聊天。Codex 在任务进行中禁用这两种操作。

### 使用 `/permissions` 更新权限

1. 输入 `/permissions` 然后按回车。
2. 选择与您舒适水平相匹配的审批预设，例如 `Auto` 以进行无干预运行或 `Read Only` 以审核编辑。

预期：Codex 宣告更新的策略。未来的操作将遵循更新的批准模式，直到您再次更改它。

### 使用 `/copy` 复制最新响应

1. 输入 `/copy` 然后按回车。

预期：Codex 将最新完成的 Codex 输出复制到您的剪贴板。

如果一个回合仍在进行中，`/copy` 将使用最新完成的输出，而不是正在进行的响应。命令在第一个完成的 Codex 输出和立即回滚后是不可用的。

您还可以在主要 TUI 中按 `Ctrl` + `O` 以便在不打开斜杠命令菜单的情况下复制最新完成的响应。

### 使用 `/sandbox-add-read-dir` 授予沙盒读取访问权限

该命令仅在 Windows 上本地运行 CLI 时可用。

1. 输入 `/sandbox-add-read-dir C:\absolute\directory\path` 然后按回车。
2. 确认路径是现有的绝对目录。

预期：Codex 刷新 Windows 沙盒策略，并授予该目录的读取访问权限，以便后续在沙盒中运行的命令。

### 使用 `/status` 检查会话状态

1. 在任何对话中，输入 `/status`。
2. 查看活动模型、批准政策、可写根目录和当前令牌使用情况的输出。

预期：您将看到类似 `codex status` 在 shell 中打印的摘要，确认 Codex 在您预期的地方运行。

### 使用 `/debug-config` 检查配置层

1. 输入 `/debug-config`。
2. 查看输出以检查配置层顺序（优先级最低先显示）、开/关状态及政策来源。

预期：Codex 打印层诊断信息以及政策详情，例如 `allowed_approval_policies`、`allowed_sandbox_modes`、`mcp_servers`、`rules`、`enforce_residency` 和 `experimental_network`。

使用此输出调试有效设置为何不同于 `config.toml`。

### 使用 `/statusline` 配置页脚项目

1. 输入 `/statusline`。
2. 使用选择器切换和重新排序项目，然后确认。

预期：页脚状态行立即更新，并永久保存在 `config.toml` 的 `tui.status_line` 中。

可用状态行项目包括模型、模型加推理、上下文统计、速率限制、git 分支、令牌计数器、会话 id、当前目录/项目根和 Codex 版本。

### 使用 `/title` 配置终端标题项目

1. 输入 `/title`。
2. 使用选择器切换和重新排序项目，然后确认。

预期：终端窗口或标签标题立即更新，并永久保存在 `config.toml` 的 `tui.terminal_title` 中。

可用标题项目包括应用名称、项目、旋转器、状态、线程、git 分支、模型和任务进度。

### 使用 `/ps` 检查后台终端

1. 输入 `/ps`。
2. 查看后台终端及其状态的列表。

预期：Codex 显示每个后台终端的命令及最多三行最近的非空输出，以便您可以一目了然地掌握进度。

当 `unified_exec` 在使用时，后台终端可见；否则，列表可能为空。

### 使用 `/stop` 停止后台终端

1. 输入 `/stop`。
2. 在 Codex 提示您之前确认是否停止列出的终端。

预期：Codex 停止当前会话的所有后台终端。`/clean` 仍然可用作 `/stop` 的别名。

### 使用 `/compact` 保持转录简洁

1. 在长时间交流后，输入 `/compact`。
2. 当 Codex 提供总结对话的选项时确认。

预期：Codex 用简洁的摘要替换早期回合，释放上下文，同时保留关键细节。

### 使用 `/diff` 查看更改

1. 输入 `/diff` 以检查看 Git 差异。
2. 在 CLI 中滚动输出，查看编辑和新增文件。

预期：Codex 显示您已暂存的更改、尚未暂存的更改以及 Git 尚未开始跟踪的文件，以便您决定保留哪些更改。

### 使用 `/mention` 高亮文件

1. 输入 `/mention`，后跟路径，例如 `/mention src/lib/api.ts`。
2. 从弹出窗口中选择匹配的结果。

预期：Codex 将文件添加到对话中，确保后续回合可以直接引用它。

### 使用 `/new` 开始新的对话

1. 输入 `/new` 然后按回车。

预期：Codex 在同一 CLI 会话中开始一个新的对话，因此您可以在不离开终端的情况下切换任务。

与 `/clear` 不同，`/new` 不会首先清除当前终端视图。

### 使用 `/resume` 恢复已保存的对话

1. 输入 `/resume` 然后按回车。
2. 从保存会话选择器中选择您想要的会话。

预期：Codex 重新加载选定对话的转录，允许您继续之前的工作，同时保持原始历史记录完整。

### 使用 `/fork` 分叉当前对话

1. 输入 `/fork` 然后按回车。

预期：Codex 将当前对话克隆到一个新线程，赋予新的 ID，而不更改原始转录，以便您可以并行探索替代方法。

如果您需要分叉已保存的会话而不是当前会话，请在终端中运行 `codex fork` 以打开会话选择器。

### 使用 `/init` 生成 `AGENTS.md`

1. 在您希望 Codex 查找永久指令的目录中运行 `/init`。
2. 审查生成的 `AGENTS.md`，然后编辑以符合您的存储库规范。

预期：Codex 创建一个 `AGENTS.md` 脚手架，您可以进一步完善并提交以供将来会话使用。

### 使用 `/review` 请求工作树审核

1. 输入 `/review`。
2. 如果您想检查确切的文件更改，请后续输入 `/diff`。

预期：Codex 总结在您的工作树中发现的问题，重点关注行为变化和缺失的测试。它使用当前会话模型，除非您在 `config.toml` 中设置了 `review_model`。

### 使用 `/mcp` 列出 MCP 工具

2. 查看列表以确认哪些 MCP 服务器和工具可用。

预期：您会看到配置的模型上下文协议 (MCP) 工具，Codex 可以在此会话中调用。

1. 输入 `/plugins`。

预期：Codex 打开插件浏览器，您可以查看已安装的插件、您的配置允许发现的插件以及已安装插件的状态。在已安装插件上按 `Space` 切换其启用状态。

### 使用 `/agent` 切换代理线程

1. 输入 `/agent` 然后按回车。
2. 从选择器中选择您想要的线程。

预期：Codex 切换活动线程以便您可以检查或继续该代理的工作。

### 使用 `/feedback` 发送反馈

1. 输入 `/feedback` 然后按回车。
2. 按照提示提供日志或诊断信息。

预期：Codex 收集请求的诊断信息并提交给维护者。

### 使用 `/logout` 注销

1. 输入 `/logout` 然后按回车。

预期：Codex 清除当前用户会话的本地凭据。

### 使用 `/quit` 或 `/exit` 退出 CLI

1. 输入 `/quit`（或 `/exit`）然后按回车。

预期：Codex 立即退出。在此之前保存或提交任何重要工作。
