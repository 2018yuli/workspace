Codex 从多个位置读取配置细节。您的个人默认值存储在 `~/.codex/config.toml` 中，您可以通过 `.codex/config.toml` 文件添加项目覆盖。出于安全原因，Codex 仅在您信任该项目时加载项目 `.codex/` 层。

## Codex 配置文件

Codex 在 `~/.codex/config.toml` 中存储用户级配置。要将设置范围限制为特定项目或子文件夹，请在您的代码仓库中添加 `.codex/config.toml` 文件。

要从 Codex IDE 扩展中打开配置文件，请选择右上角的齿轮图标，然后选择 **Codex 设置 > 打开 config.toml**。

CLI 和 IDE 扩展共享相同的配置层。您可以使用它们来：

- 设置默认模型和提供程序。
- 配置 [批准政策和沙盒设置](https://developers.openai.com/codex/agent-approvals-security#sandbox-and-approvals)。
- 配置 [MCP 服务器](https://developers.openai.com/codex/mcp)。

## 配置优先级

Codex 按以下顺序解析值（优先级最高的在前）：

1. CLI 标志和 `--config` 覆盖
2. [配置文件](https://developers.openai.com/codex/config-advanced#profiles) 值（来自 `--profile <name>`）
3. 项目配置文件：`.codex/config.toml`，从项目根向下到您当前的工作目录（离得近者优先；仅限受信任项目）
4. 用户配置：`~/.codex/config.toml`
5. 系统配置（如果存在）：Unix 上的 `/etc/codex/config.toml`
6. 内置默认值

使用该优先级在顶层设置共享默认值，并保持配置文件集中于不同值。

如果您将项目标记为不受信任，Codex 将跳过项目范围 `.codex/` 层，包括项目本地配置、钩子和规则。用户和系统配置仍将加载，包括用户/全局钩子和规则。

有关通过 `-c`/`--config` 提供一次性覆盖（包括 TOML 引用规则），请参阅 [高级配置](https://developers.openai.com/codex/config-advanced#one-off-overrides-from-the-cli)。

在托管机器上，您的组织可能还会通过 `requirements.toml` 强制执行约束（例如，不允许 `approval_policy = "never"` 或 `sandbox_mode = "danger-full-access"`）。请参阅 [托管配置](https://developers.openai.com/codex/enterprise/managed-configuration) 和 [管理员强制要求](https://developers.openai.com/codex/enterprise/managed-configuration#admin-enforced-requirements-requirementstoml)。

## 常见配置选项

以下是人们最常更改的一些选项：

#### 默认模型

选择 Codex 默认使用的模型，在 CLI 和 IDE 中。

```
model = "gpt-5.5"
```

#### 批准提示

控制 Codex 在执行生成的命令之前何时暂停提问。

```
approval_policy = "on-request"
```

有关 `untrusted`、`on-request` 和 `never` 之间行为差异的更多信息，请参见 [无批准提示运行](https://developers.openai.com/codex/agent-approvals-security#run-without-approval-prompts) 和 [常见沙盒与批准组合](https://developers.openai.com/codex/agent-approvals-security#common-sandbox-and-approval-combinations)。

#### 沙盒级别

调整 Codex 在执行命令时可以进行的文件系统和网络访问量。

```
sandbox_mode = "workspace-write"
```

有关每种模式的行为（包括受保护的 `.git`/`.codex` 路径和网络默认值），请参见 [沙盒与批准](https://developers.openai.com/codex/agent-approvals-security#sandbox-and-approvals)、[可写根中的受保护路径](https://developers.openai.com/codex/agent-approvals-security#protected-paths-in-writable-roots) 和 [网络访问](https://developers.openai.com/codex/agent-approvals-security#network-access)。

#### Windows 沙盒模式

在 Windows 上原生运行 Codex 时，在 `windows` 表中将原生沙盒模式设置为 `elevated`。仅在没有管理权限或提升设置失败时使用 `unelevated`。

```
[windows]
sandbox = "elevated"   # 推荐
# sandbox = "unelevated" # 如果缺少管理员权限/设置失败则作为后备
```

#### 网络搜索模式

Codex 默认启用本地任务的网络搜索，并从网络搜索缓存中提供结果。缓存是 OpenAI 维护的网络结果索引，因此缓存模式返回预先索引的结果，而不是获取实时页面。这减少了来自任意实时内容的提示注入的暴露，但您仍应将网络结果视为不可信。如果您使用 `--yolo` 或其他 [完全访问沙盒设置](https://developers.openai.com/codex/agent-approvals-security#common-sandbox-and-approval-combinations)，网络搜索默认会返回实时结果。选择带有 `web_search` 的模式：

- `"cached"`（默认）从网络搜索缓存中提供结果。
- `"live"` 从网络获取最新数据（相同于 `--search`）。
- `"disabled"` 关闭网络搜索工具。

```
web_search = "cached"  # 默认；从网络搜索缓存中提供结果
# web_search = "live"  # 从网络获取最新数据（相同于 --search）
# web_search = "disabled"
```

#### 推理努力

调整模型在支持时施加的推理努力程度。

```
model_reasoning_effort = "high"
```

#### 交流风格

为支持的模型设置默认交流风格。

```
personality = "friendly" # 或 "pragmatic" 或 "none"
```

您可以在活动会话中使用 `/personality` 覆盖此设置，或在使用应用服务器 API 时按线程/轮次进行覆盖。

#### 命令环境

控制 Codex 转发到生成命令的环境变量。

```
[shell_environment_policy]
include_only = ["PATH", "HOME"]
```

#### 日志目录

覆盖 Codex 写入本地日志文件（如 `codex-tui.log`）的位置。

```
log_dir = "/absolute/path/to/codex-logs"
```

对于一次性运行，您也可以从 CLI 设置：

```
codex -c log_dir=./.codex-log
```

## 功能标志

在 `config.toml` 中使用 `[features]` 表来切换可选和实验性功能。

```
[features]
shell_snapshot = true           # 加快重复命令的速度
```

### 支持的功能

| 键 | 默认 | 成熟度 | 描述 |
| --- | --- | --- | --- |
| `apps` | false | 实验性 | 启用 ChatGPT 应用/连接器支持 |
| `codex_hooks` | true | 稳定 | 启用来自 `hooks.json` 或内联 `[hooks]` 的生命周期钩子。请参见 [钩子](https://developers.openai.com/codex/hooks)。 |
| `fast_mode` | true | 稳定 | 启用快速模式选择和 `service_tier = "fast"` 路径 |
| `memories` | false | 稳定 | 启用 [记忆](https://developers.openai.com/codex/memories) |
| `multi_agent` | true | 稳定 | 启用子代理协作工具 |
| `personality` | true | 稳定 | 启用个性选择控件 |
| `shell_snapshot` | true | 稳定 | 快照你的 shell 环境以加快重复命令的速度 |
| `shell_tool` | true | 稳定 | 启用默认的 `shell` 工具 |
| `unified_exec` | `true` 除了 Windows | 稳定 | 使用统一的基于 PTY 的执行工具 |
| `undo` | false | 稳定 | 通过每轮 git 影子快照启用撤销 |
| `web_search` | true | 不推荐 | 过时的切换；请使用顶级 `web_search` 设置 |
| `web_search_cached` | false | 不推荐 | 映射到 `web_search = "cached"` 的过时切换，当未设置时为真 |
| `web_search_request` | false | 不推荐 | 映射到 `web_search = "live"` 的过时切换，当未设置时为真 |

成熟度列使用功能成熟度标签，如实验性、测试版和稳定版。有关如何解释这些标签的信息，请参阅 [功能成熟度](https://developers.openai.com/codex/feature-maturity)。

省略功能键以保持其默认值。

有关当前生命周期钩子 MVP，请参见 [钩子](https://developers.openai.com/codex/hooks)。

### 启用功能

- 在 `config.toml` 中，在 `[features]` 下添加 `feature_name = true`。
- 从 CLI 运行 `codex --enable feature_name`。
- 要启用多个功能，请运行 `codex --enable feature_a --enable feature_b`。
- 要禁用某个功能，请在 `config.toml` 中将其键设置为 `false`。
