## Codex 云任务的运行方式

提交任务时发生以下情况：

1. Codex 创建一个容器并在选择的分支或提交 SHA 上检出您的 repo。
2. Codex 运行您的设置脚本，缓存容器恢复时还会运行可选的维护脚本。
3. Codex 应用您的互联网访问设置。设置脚本在有互联网访问的情况下运行。代理的互联网访问默认关闭，但您可以在需要时启用有限或不受限制的访问。请参见[代理互联网访问](https://developers.openai.com/codex/cloud/internet-access)。
4. 代理在循环中运行终端命令。它编辑代码，运行检查，并尝试验证其工作。如果您的 repo 包含 `AGENTS.md`，代理将使用它寻找特定项目的 lint 和测试命令。
5. 当代理完成时，它会显示其答案和它更改的任何文件的差异。您可以打开 PR 或提问后续问题。

## 默认通用镜像

Codex 代理在一个名为 `universal` 的默认容器镜像中运行，该镜像预装了常见的语言、包和工具。

在环境设置中，选择 **设置包版本** 以固定 Python、Node.js 和其他运行时的版本。

有关安装内容的详细信息，请参见
[openai/codex-universal](https://github.com/openai/codex-universal) 的
参考 Dockerfile 和可拉取和本地测试的镜像。

虽然 `codex-universal` 为速度和便利性预装了语言，但您也可以使用[设置脚本](https://developers.openai.com/codex/cloud/environments#manual-setup)向容器安装额外包。

## 环境变量和秘密

**环境变量** 在任务的整个持续时间内设置（包括设置脚本和代理阶段）。

**秘密** 与环境变量类似，但：

- 它们以额外的加密层存储，仅在任务执行时被解密。
- 它们仅对设置脚本可用。出于安全原因，秘密在代理阶段开始前被移除。

## 自动设置

对于使用常见包管理器（`npm`、`yarn`、`pnpm`、`pip`、`pipenv` 和 `poetry`）的项目，Codex 可以自动安装依赖项和工具。

## 手动设置

如果您的开发设置更复杂，您也可以提供自定义设置脚本。例如：

```
# 安装类型检查器
pip install pyright

# 安装依赖项
poetry install --with test
pnpm install

```

设置脚本在与代理不同的 Bash 会话中运行，因此像 `export` 这样的命令不会持续到代理阶段。要持久化环境变量，请将其添加到 `~/.bashrc` 或在环境设置中配置。

## 容器缓存

Codex 最多缓存容器状态 12 小时，以加速新任务和后续任务。

当环境被缓存时：

- Codex 克隆库并检出默认分支。
- Codex 运行设置脚本，并缓存结果容器状态。

当恢复缓存的容器时：

- Codex 检出任务指定的分支。
- Codex 运行维护脚本（可选）。当设置脚本在较旧提交上运行且依赖项需要更新时，这很有用。

如果您更改设置脚本、维护脚本、环境变量或秘密，Codex 会自动使缓存无效。如果您的 repo 发生更改，使缓存状态不兼容，请在环境页面上选择 **重置缓存**。

对于商业和企业用户，缓存在所有有权访问该环境的用户之间共享。失效缓存将影响您工作区中该环境的所有用户。

## 互联网访问和网络代理

在设置脚本阶段，允许互联网访问以安装依赖项。在代理阶段，互联网访问默认关闭，但您可以配置有限或不受限制的访问。请参见[代理互联网访问](https://developers.openai.com/codex/cloud/internet-access)。

环境在 HTTP/HTTPS 网络代理后运行，以出于安全和防止滥用的目的。所有出站互联网流量都经过此代理。
