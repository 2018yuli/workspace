在 Linear 中使用 Codex 将工作委派给问题。将一项问题分配给 Codex 或在评论中提到 `@Codex`，Codex 创建一个云任务并回复进展和结果。

Linear 中的 Codex 可在付费套餐中使用（请参见 [定价](https://developers.openai.com/codex/pricing)）。

如果您是企业计划的用户，请请求您的 ChatGPT 工作区管理员在 [工作区设置](https://chatgpt.com/admin/settings) 中开启 Codex 云任务，并在 [连接设置](https://chatgpt.com/admin/ca) 中启用 **Codex for Linear**。

## 设置 Linear 集成

1. 通过在 [Codex](https://chatgpt.com/codex) 中连接 GitHub 并为您希望 Codex 工作的仓库创建一个 [环境](https://developers.openai.com/codex/cloud/environments)，设置 [Codex 云任务](https://developers.openai.com/codex/cloud)。
2. 前往 [Codex 设置](https://chatgpt.com/codex/settings/connectors)，为您的工作区安装 **Codex for Linear**。
3. 通过在 Linear 问题的评论线程中提到 `@Codex` 来链接您的 Linear 帐户。

## 将工作委派给 Codex

您可以通过两种方式进行委派：

### 将问题分配给 Codex

在安装集成后，您可以像将问题分配给队友一样将问题分配给 Codex。Codex 开始工作并向问题发布更新。

![将 Codex 分配给 Linear 问题（亮模式）](https://developers.openai.com/images/codex/integrations/linear-assign-codex-light.webp)![将 Codex 分配给 Linear 问题（暗模式）](https://developers.openai.com/images/codex/integrations/linear-assign-codex-dark.webp)

### 在评论中提到 `@Codex`

您还可以在评论线程中提到 `@Codex` 来委派工作或提出问题。在 Codex 回复后，请在线程中继续跟进，以保持同一会话。

![在 Linear 问题评论中提到 Codex（亮模式）](https://developers.openai.com/images/codex/integrations/linear-comment-light.webp)![在 Linear 问题评论中提到 Codex（暗模式）](https://developers.openai.com/images/codex/integrations/linear-comment-dark.webp)

当 Codex 开始处理一个问题时，它会选择一个 [环境和仓库](https://developers.openai.com/codex/integrations/linear#how-codex-chooses-an-environment-and-repo) 进行工作。
要固定特定的仓库，请在评论中包括它，例如：`@Codex fix this in openai/codex`。

要跟踪进展：

- 打开问题上的 **活动** 查看进度更新。
- 打开任务链接以获取更详细的信息。

当任务完成时，Codex 会发布摘要和已完成任务的链接，以便您创建拉取请求。

### Codex 如何选择环境和仓库

- Linear 根据问题上下文建议一个仓库。Codex 选择与该建议最匹配的环境。如果请求模糊，它将回退到您最近使用的环境。
- 任务在该环境的仓库映射中列出的第一个仓库的默认分支上运行。如果您需要不同的默认分支或更多的仓库，请在 Codex 中更新仓库映射。
- 如果没有合适的环境或仓库可用，Codex 将在 Linear 中回复说明如何解决问题，然后再试一次。

## 自动将问题分配给 Codex

您可以使用分诊规则自动将问题分配给 Codex：

1. 在 Linear 中，转到 **设置**。
2. 在 **您的团队** 下，选择您的团队。
3. 在工作流设置中，打开 **分诊** 并将其开启。
4. 在 **分诊规则** 中，创建一个规则并选择 **委派** \> **Codex**（以及您希望设置的任何其他属性）。

Linear 会自动将进入分诊的新问题分配给 Codex。
当您使用分诊规则时，Codex 使用问题创建者的帐户运行任务。

![示例分诊规则将所有内容分配给 Codex 并将其标记为“分诊”状态的屏幕截图（亮模式）](https://developers.openai.com/images/codex/integrations/linear-triage-rule-light.webp)![示例分诊规则将所有内容分配给 Codex 并将其标记为“分诊”状态的屏幕截图（暗模式）](https://developers.openai.com/images/codex/integrations/linear-triage-rule-dark.webp)

## 数据使用、隐私和安全

当您提到 `@Codex` 或将问题分配给它时，Codex 会接收您的问题内容以理解您的请求并创建任务。
数据处理遵循 OpenAI 的 [隐私政策](https://openai.com/privacy)、[使用条款](https://openai.com/terms/) 以及其他适用的 [政策](https://openai.com/policies)。
有关安全性的更多信息，请参见 [Codex 安全文档](https://developers.openai.com/codex/agent-approvals-security)。

Codex 使用大型语言模型，这些模型可能会出错。请始终检查答案和差异。

## 提示和故障排除

- **缺少连接**：如果 Codex 无法确认您的 Linear 连接，它会在问题中回复一个链接，让您连接帐户。
- **意外的环境选择**：在线程中回复您想要的环境（例如，`@Codex please run this in openai/codex`）。
- **代码的错误部分**：在问题中添加更多上下文，或在您的 `@Codex` 评论中给出明确指示。
- **更多帮助**：请查看 [OpenAI 帮助中心](https://help.openai.com/)。

## 连接 Linear 进行本地任务（MCP）

如果您正在使用 Codex 应用、CLI 或 IDE 扩展，并希望 Codex 访问本地的 Linear 问题，请配置 Codex 使用 Linear 模型上下文协议（MCP）服务器。

要了解更多，请 [查看 Linear MCP 文档](https://linear.app/integrations/codex-mcp)。

MCP 服务器的设置步骤在您使用 IDE 扩展或 CLI 时是相同的，因为两者共享相同的配置。

### 使用 CLI（推荐）

如果您已安装 CLI，请运行：

```
codex mcp add linear --url https://mcp.linear.app/mcp


```

这会提示您使用您的 Linear 帐户登录，并将其连接到 Codex。

### 手动配置

1. 在编辑器中打开 `~/.codex/config.toml`。
2. 添加以下内容：

```
[mcp_servers.linear]
url = "https://mcp.linear.app/mcp"


```

3. 运行 `codex mcp login linear` 以登录。
