# 治理和可观察性

Codex 为企业团队提供了对采用情况和影响的可见性，以及安全和合规程序所需的可审核性。使用自助仪表板进行日常跟踪，使用分析 API 进行程序化报告，使用合规 API 将详细日志导出到你的治理堆栈中。

## 跟踪 Codex 使用情况的方法

有三种方法可以监控 Codex 使用情况，具体取决于你的需求：

- **分析仪表板**：快速查看采用情况和代码审查影响。
- **分析 API**：将结构化的日常指标导入到你的数据仓库或 BI 工具中。
- **合规 API**：导出详细的活动日志以进行审核、监控和调查。

## 分析仪表板

![Codex 分析仪表板](https://developers.openai.com/images/codex/enterprise/analytics.png)

### 仪表板

[分析仪表板](https://chatgpt.com/codex/settings/analytics) 允许 ChatGPT 工作区管理员跟踪功能采用情况。

Codex 提供以下仪表板：

- 按产品（CLI，IDE，云，代码审查）每日用户
- 每日代码审查用户
- 每日代码审查
- 按优先级级别的代码审查
- 按反馈情感的每日代码审查
- 每日云任务
- 每日云用户
- 每日 VS Code 扩展用户
- 每日 CLI 用户

### 数据导出

管理员还可以将 Codex 分析数据导出为 CSV 或 JSON 格式。Codex 提供以下导出选项：

- 代码审查用户和审查（每日独特用户和已完成的总审查）
- 代码审查发现和反馈（每日评论、反应、回复和优先级发现的数量）
- 云用户和任务（每日独特云用户和已完成的任务）
- CLI 和 VS Code 用户（每日独特的 Codex CLI 和 VS Code 扩展用户）
- 每用户的会话和消息（每个 Codex 用户在各表面上的每日会话开始数和用户消息数量）

## 分析 API

当你需要自动化报告、构建内部仪表板或将 Codex 指标与现有工程数据结合使用时，请使用 [分析 API](https://chatgpt.com/codex/settings/apireference)。

### 测量内容

分析 API 提供工作区的每日时间序列指标，带有每用户细分和每客户端使用情况的可选细分。

### 端点

#### 每日使用情况和采用情况

- 线程、回合和积分的每日总数
- 按客户端表面细分
- 可选的每用户报告，用于采用情况和高阶用户分析

#### 代码审查活动

- Codex 完成的拉取请求审查
- Codex 生成的总评论
- 评论的严重程度细分

#### 用户与代码审查的互动

- 回复 Codex 评论
- 反应，包括点赞和点踩
- 团队如何响应 Codex 反馈的互动细分

### 工作原理

分析是每日和时间窗口化的。结果是按时间顺序排列的，并以光标分页的形式返回。你可以按工作区查询，并可选择按用户分组或按工作区级别汇总。

### 常见用例

- 工程可观察性仪表板
- 领导更新的采用报告
- 使用治理和成本监控

## 合规 API

当你需要针对安全、法律和治理工作流程的可审核记录时，请使用 [合规 API](https://chatgpt.com/admin/api-reference)。

### 测量内容

合规 API 为企业提供了一种导出 Codex 活动日志和元数据的方法，以便你可以将这些数据连接到现有的审核、监控和安全工作流程。它旨在与 eDiscovery、DLP、SIEM 或其他合规系统等工具配合使用。

对于通过 ChatGPT 认证的 Codex 使用情况，合规 API 导出提供 Codex 活动的审核记录，并可用于调查和合规工作流程。这些审核日志保留长达 30 天。通过 API 密钥认证的 Codex 使用情况遵循你的 API 组织设置，不包含在合规 API 导出中。

### 可导出的内容

#### 活动日志

- 发送给 Codex 的提示文本
- Codex 生成的响应
- 标识符，如工作区、用户、时间戳和模型
- 令牌使用量和相关请求元数据

#### 审核和调查的元数据

使用记录元数据回答以下问题：

- 谁运行了任务
- 何时运行
- 使用了哪个模型
- 处理了多少内容

#### 常见用例

- 安全调查
- 合规报告
- 政策执行审核
- 将事件路由到 SIEM 和 eDiscovery 管道

### 不提供的内容

- 生成的代码行（这是生产力的一个噪音代理，可能会激励错误行为）
- 建议的接受率（几乎是 100%，因为用户通常首先接受变更）
- 代码质量或性能 KPI

## 推荐模式

大多数企业结合使用以下几种：

1. **分析仪表板** 进行自助监控和快速答案
2. **分析 API** 进行自动化报告和 BI 集成
3. **合规 API** 进行审核导出和调查
