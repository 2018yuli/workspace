## 提示

你通过发送提示（用户消息）与 Codex 互动，这些提示描述你希望它做什么。

示例提示：

```
解释一下转换模块是如何工作的，以及其他模块如何使用它。
```

```
添加一个新的命令行选项 `--json` 来输出 JSON。
```

当你提交一个提示时，Codex 在一个循环中工作：它调用模型，然后执行模型输出所指示的操作，例如文件读取、文件编辑和工具调用。此过程在任务完成或你取消它时结束。

与 ChatGPT 一样，Codex 的有效性取决于你给它的指令。以下是一些我们在提示 Codex 时发现的有用提示：

- 当 Codex 能够验证其工作时，它会产生更高质量的输出。包含重现问题的步骤、验证功能以及运行 linting 和提交前检查。
- 当你将复杂工作分解成更小、更专注的步骤时，Codex 处理得更好。较小的任务更易于 Codex 测试，也更容易让你审核。如果你不确定如何拆分任务，可以请 Codex 提出一个计划。

有关提示 Codex 的更多想法，请参考 [工作流](https://developers.openai.com/codex/workflows)。

## 线程

线程是一个单一会话：你的提示加上随后模型输出和工具调用。一个线程可以包含多个提示。例如，你的第一个提示可能要求 Codex 实现一个功能，而后续提示可能要求它添加测试。

当 Codex 正在积极处理时，线程被称为“运行中”。你可以同时运行多个线程，但应避免让两个线程修改同一文件。你也可以稍后通过另一个提示继续运行一个线程。

线程可以在本地或云中运行：

- **本地线程** 在你的机器上运行。Codex 可以读取和编辑你的文件并运行命令，因此你可以看到所做的更改并使用你现有的工具。为了降低在工作区之外意外更改的风险，本地线程在 [沙箱](https://developers.openai.com/codex/agent-approvals-security) 中运行。
- **云线程** 在一个隔离的 [环境](https://developers.openai.com/codex/cloud/environments) 中运行。Codex 克隆你的代码库，并检出它正在工作的分支。当你希望并行运行工作或从另一个设备委派任务时，云线程非常有用。要使用云线程与你的代码库，首先将代码推送到 GitHub。你还可以 [从本地机器委派任务](https://developers.openai.com/codex/ide/cloud-tasks)，包括你当前的工作状态。

在 Codex 应用中，你还可以在未选择项目的情况下启动聊天。聊天不与保存的代码库或项目文件夹相关联。将它们用于研究、规划、连接工具工作流或其他不应从代码库开始的工作。聊天使用 Codex 管理的 `threads` 目录作为其工作位置。默认情况下，该位置为 `~/.codex/threads`。要更改此状态的基础位置，请设置 `CODEX_HOME`；请参阅 [配置和状态位置](https://developers.openai.com/codex/config-advanced#config-and-state-locations)。

## 上下文

当你提交一个提示时，包含 Codex 可以使用的上下文，例如对相关文件和图像的引用。Codex IDE 扩展自动包括打开文件的列表和选定文本范围作为上下文。

当代理工作时，它还会从文件内容、工具输出和它所做的事情及仍需做的事情的持续记录中收集上下文。

线程中的所有信息必须适合模型的 **上下文窗口**，这个窗口因模型而异。Codex 会监控并报告剩余空间。对于较长的任务，Codex 可能会通过总结相关信息并丢弃不相关的细节来自动 **压缩** 上下文。通过重复压缩，Codex 可以在多个步骤中继续处理复杂任务。
