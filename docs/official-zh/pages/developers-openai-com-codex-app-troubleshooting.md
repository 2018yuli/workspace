## 常见问题

### 文件出现在侧边栏中，而 Codex 并未编辑

如果你的项目在 Git 仓库内，审查面板会自动显示基于项目的 Git 状态的更改，包括 Codex 未进行的更改。

在审查面板中，你可以在暂存的更改和未暂存的更改之间切换，并比较你的分支与主分支。

如果你只想查看最后一次 Codex 操作的更改，请将差异面板切换到“最后一次更改”视图。

[了解更多关于如何使用审查面板的信息](https://developers.openai.com/codex/app/review)。

### 从侧边栏移除项目

要从侧边栏移除项目，将鼠标悬停在项目名称上，点击三个点并选择“移除”。要恢复它，请使用 **添加新项目** 按钮或使用 `Cmd` + `O` 重新添加项目。

### 找到归档的线程

归档的线程可以在 [设置](codex://settings) 中找到。当你取消归档线程时，它将重新出现在侧边栏的原始位置。

### 只有一些线程出现在侧边栏

侧边栏允许根据项目的状态过滤线程。如果你缺少线程，请点击 **线程** 标签旁边的过滤图标并切换到按时间顺序排列。如果仍然没有看到线程，请打开 [设置](codex://settings) 并检查归档聊天或归档线程部分。

### 代码在工作树上无法运行

工作树是在不同目录中创建的，仅继承已检查到 Git 的文件。根据你管理项目的依赖项和工具的方式，你可能需要在工作树上运行一些设置脚本，使用 [本地环境](https://developers.openai.com/codex/app/local-environments)。或者，你可以在常规本地项目中查看更改。查看 [工作树文档](https://developers.openai.com/codex/app/worktrees) 以了解更多信息。

### 应用不接收队友共享的本地环境

本地环境配置必须位于项目根目录的 `.codex` 文件夹中。如果你在一个包含多个项目的单一代码库中工作，请确保在包含 `.codex` 文件夹的目录中打开项目。

### Codex 请求访问 Apple Music

根据你的任务，Codex 可能需要浏览文件系统。macOS 上的某些目录，包括音乐、下载或桌面，需要用户额外批准。如果 Codex 需要读取你的主目录，macOS 会提示你批准访问这些文件夹。

### 自动化创建许多工作树

频繁的自动化操作可能会随着时间的推移创建许多工作树。归档你不再需要的自动化操作，避免固定运行，除非你打算保留它们的工作树。

### 选择错误目标后恢复提示

如果你意外以错误的目标 (**本地**、**工作树**或 **云**) 开始线程，可以取消当前运行并通过在创作界面按上箭头键来恢复之前的提示。

### 功能在 Codex CLI 中正常工作，但在 Codex 应用程序中不工作

Codex 应用程序和 Codex CLI 使用相同的基础 Codex 代理和配置，但可能随时依赖于不同版本的代理，并且某些实验性功能可能首先只在 Codex CLI 中可用。

要获取你系统上 Codex CLI 的版本，请运行：

```
codex --version
```

要获取与 Codex 应用程序捆绑的 Codex 版本，请运行：

```
/Applications/Codex.app/Contents/Resources/codex --version
```

## 反馈和日志

在消息创作界面输入 `/` 以提供反馈。如果你在现有对话中触发反馈，可以选择与反馈一起分享当前会话。提交反馈后，你将收到一个会话 ID，可以与团队共享。

要报告问题：

1. 在 Codex GitHub 仓库中查找 [现有问题](https://github.com/openai/codex/issues)。
2. [打开一个新的 GitHub 问题](https://github.com/openai/codex/issues/new?template=2-bug-report.yml&steps=Uploaded%20thread%3A%20019c0d37-d2b6-74c0-918f-0e64af9b6e14)

更多日志可在以下位置获取：

- 应用日志 (macOS): `~/Library/Logs/com.openai.codex/YYYY/MM/DD`
- 会话记录: `$CODEX_HOME/sessions` (默认: `~/.codex/sessions`)
- 归档会话: `$CODEX_HOME/archived_sessions` (默认: `~/.codex/archived_sessions`)

如果你共享日志，请先检查它们以确认不包含敏感信息。

## 卡住状态和恢复模式

如果线程似乎卡住：

1. 检查 Codex 是否在等待批准。
2. 打开终端并运行基本命令，如 `git status`。
3. 使用更小、更专注的提示开始新线程。

如果你误取消了工作树创建并失去了提示，请在创作界面按上箭头键以恢复它。

## 终端问题

**终端似乎卡住**

1. 关闭终端面板。
2. 使用 `Cmd` + `J` 重新打开它。
3. 重新运行基本命令，如 `pwd` 或 `git status`。

如果命令的行为与预期不同，首先验证终端中的当前目录和分支。

如果仍然卡住，请等待你活动的 Codex 线程完成，然后重启应用程序。

**字体未正确渲染**

Codex 在审查面板、集成终端和应用程序内显示的任何其他代码中使用相同的字体。你可以在 [设置](codex://settings) 面板中将其配置为 **代码字体**。
