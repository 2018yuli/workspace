## 斜杠命令

斜杠命令让您在不离开线程编辑器的情况下控制 Codex。可用的命令根据您的环境和访问权限而异。

### 使用斜杠命令

1. 在线程编辑器中输入 `/`。
2. 从列表中选择一个命令，或继续输入进行过滤（例如，`/status`）。

您还可以通过在线程编辑器中输入 `$` 明确调用技能。请参见 [技能](https://developers.openai.com/codex/skills)。

启用的技能也会出现在斜杠命令列表中。

### 可用的斜杠命令

| 斜杠命令 | 描述 |
| --- | --- |
| `/feedback` | 打开反馈对话框提交反馈，并可选择性地包括日志。 |
| `/mcp` | 打开 MCP 状态以查看连接的服务器。 |
| `/plan-mode` | 切换多步计划的计划模式。 |
| `/review` | 开始代码审查模式，审查未提交的更改或与基础分支比较。 |
| `/status` | 显示线程 ID、上下文使用情况和速率限制。 |

## 深度链接

Codex 应用注册 `codex://` URL 方案，以便链接可以直接打开应用的特定部分。

| 深度链接 | 打开 | 支持的查询参数 |
| --- | --- | --- |
| `codex://settings` | 设置。 | 无。 |
| `codex://skills` | 技能。 | 无。 |
| `codex://automations` | 自动化创建模式中的收件箱。 | 无。 |
| `codex://threads/<thread-id>` | 本地线程。`<thread-id>` 必须是 UUID。 | 无。 |
| `codex://new` | 新线程。 | 可选：`prompt`，`originUrl`，`path`。 |

对于新线程深度链接：

- `prompt` 设置初始编辑器文本。
- `path` 必须是本地目录的绝对路径，并且在有效时，该目录将成为新线程的活动工作区。
- `originUrl` 尝试通过 Git 远程 URL 匹配当前工作区根目录之一。如果同时存在 `path` 和 `originUrl`，Codex 首先解析 `path`。
