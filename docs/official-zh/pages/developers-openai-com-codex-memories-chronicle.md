## 启用 Chronicle

1. 打开 Codex 应用程序中的设置。
2. 转到 **个性化** 并确保启用了 **记忆**。
3. 在记忆设置下启用 **Chronicle**。
4. 查看同意对话框并选择 **继续**。
5. 当提示时授予 macOS 屏幕录制和辅助功能权限。
6. 设置完成后，选择 **试用** 或开始新线程。

如果 macOS 报告屏幕录制或辅助功能权限被拒绝，请打开系统设置 > 隐私与安全 > 屏幕录制或辅助功能并启用 Codex。如果 macOS 或您的组织限制了某项权限，Chronicle 将在解除限制并且 Codex 获得所需权限后开始。

## 随时暂停或禁用 Chronicle

您可以控制 Chronicle 使用屏幕上下文生成记忆的时间。使用 Codex 菜单栏图标选择 **暂停 Chronicle** 或 **恢复 Chronicle**。在会议前或查看您不希望 Codex 作为上下文使用的敏感内容时，请暂停 Chronicle。要禁用 Chronicle，请返回到 **设置 >** **个性化 > 记忆** 并关闭 **Chronicle**。

您还可以控制记忆是否在特定线程中使用。[了解更多](https://developers.openai.com/codex/memories#control-memories-per-thread)。

## 速率限制

Chronicle 通过在后台运行沙箱代理来生成从捕获的屏幕图像中生成记忆。这些代理当前会迅速消耗速率限制。

## 隐私与安全

Chronicle 使用屏幕捕获，可能包括您屏幕上可见的敏感信息。它不具备访问您的麦克风或系统音频的权限。在未获得他人同意之前，请勿使用 Chronicle 记录会议或与他人的通讯。查看您不希望记忆中的内容时，请暂停 Chronicle。

### Chronicle 将我的数据存储在哪里？

屏幕捕获是短暂的，只会在您的计算机上临时保存。临时屏幕捕获文件可能会在 Chronicle 运行时出现在 `$TMPDIR/chronicle/screen_recording/` 下。Chronicle 运行时，超过 6 小时的屏幕捕获将被删除。

Chronicle 生成的记忆与其他 Codex 记忆一样：未加密的 markdown 文件，您可以读取并根据需要进行修改。您还可以要求 Codex 搜索它们。如果您希望 Codex 忘记某些内容，可以删除文件夹中的相应文件，或选择性编辑 markdown 文件以去除您想删除的信息。您不应该手动添加新信息。生成的 Chronicle 记忆保存在您计算机的 `$CODEX_HOME/memories_extensions/chronicle/` 下（通常是 `~/.codex/memories_extensions/chronicle`）。

您屏幕捕获和记忆的两个目录可能包含敏感信息。确保您不与他人共享内容，并意识到您计算机上的其他程序也可以访问这些文件。

### 什么数据会与 OpenAI 分享？

Chronicle 在本地捕获屏幕上下文，然后定期使用 Codex 将最近活动总结为记忆。为了生成这些记忆，Chronicle 启动了一个短暂的 Codex 会话，该会话访问此屏幕上下文。该会话可能处理选定的屏幕截图帧、从截图中提取的 OCR 文本、时间信息以及相关时间窗口的本地文件路径。

用于记忆生成的屏幕捕获暂时存储在您的设备上。它们会在我们的服务器上进行处理以生成记忆，然后存储在本地设备上。除非法律要求，我们不会将截图存储在我们的服务器上，也不会将其用于训练。

生成的记忆是 Markdown 文件，存储在本地 `$CODEX_HOME/memories_extensions/chronicle/` 下。当 Codex 在未来的会话中使用记忆时，相关的记忆内容可能会作为上下文包含在该会话中，并可能在您的 ChatGPT 设置允许的情况下用于改进我们的模型。[了解更多](https://help.openai.com/en/articles/7730893-data-controls-faq)。

## 提示注入风险

使用 Chronicle 会增加来自屏幕内容的提示注入攻击的风险。例如，如果您浏览一个含有恶意代理指令的网站，Codex 可能会遵循那些指令。

## 故障排除

### 如何启用 Chronicle？

如果您未看到 Chronicle 设置，请确保您正在使用包含 Chronicle 的 Codex 应用程序构建，并且在设置中启用了记忆。

Chronicle 目前仅对 macOS 的 ChatGPT Pro 订阅者可用。
Chronicle 未在欧盟、英国和瑞士提供。

如果设置未完成：

1. 确认 Codex 拥有屏幕录制和辅助功能权限。
2. 退出并重新打开 Codex 应用程序。
3. 打开 **设置 > 个性化** 并检查 Chronicle 状态。

### 用于生成 Chronicle 记忆的模型是哪种？

Chronicle 使用与您的其他 [记忆](https://developers.openai.com/codex/memories) 相同的模型。如果您未配置特定模型，则使用您的默认 Codex 模型。要选择特定模型，请在您的 [配置](https://developers.openai.com/codex/config-basic) 中更新 `consolidation_model`。

```
[memories]
consolidation_model = "gpt-5.4-mini"
