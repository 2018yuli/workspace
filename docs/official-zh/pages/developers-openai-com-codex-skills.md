使用代理技能扩展 Codex 的任务特定功能。技能打包了指令、资源和可选脚本，以便 Codex 能够可靠地遵循工作流。技能基于 [开放代理技能标准](https://agentskills.io/)。

技能是可重用工作流的作者格式。插件是 Codex 中可重用技能和应用的可安装分发单元。使用技能设计工作流本身，然后当你希望其他开发者安装时，将其打包为 [插件](https://developers.openai.com/codex/plugins/build)。

技能可在 Codex CLI、IDE 插件和 Codex 应用中使用。

技能使用 **逐步披露** 来有效管理上下文：Codex 从每个技能的名称、描述和文件路径开始。Codex 只有在决定使用某个技能时才会加载完整的 `SKILL.md` 指令。

Codex 包含一个初始的可用技能列表，以便它可以为任务选择正确的技能。为避免挤占其余提示，此列表的数量限制在模型上下文窗口的约 2% 处，或在上下文窗口未知时为 8,000 个字符。如果安装了许多技能，Codex 首先会缩短技能描述。对于非常大的技能集，某些技能可能会从初始列表中省略，并且 Codex 会显示警告。

该预算仅适用于初始技能列表。当 Codex 选择技能时，它仍会阅读该技能的完整 SKILL.md 指令。

技能是一个包含 `SKILL.md` 文件以及可选脚本和引用的目录。`SKILL.md` 文件必须包括 `name` 和 `description`。

- my-skill/

  - SKILL.md  必需：指令 + 元数据

  - scripts/  可选：可执行代码

  - references/  可选：文档

  - assets/  可选：模板、资源

  - agents/

    - openai.yaml  可选：外观和依赖项

## Codex 如何使用技能

Codex 可以通过两种方式激活技能：

1. **显式调用：** 直接在提示中包含技能。在 CLI/IDE 中，运行 `/skills` 或输入 `$` 来提及一个技能。
2. **隐式调用：** 当任务与技能 `description` 匹配时，Codex 可以选择技能。

由于隐式匹配依赖于 `description`，请编写简洁的描述，明确范围和边界。将关键用例和触发词前置，以便 Codex 在描述缩短的情况下仍能匹配该技能。

## 创建技能

首先使用内置的创建器：

```
$skill-creator
```

创建器会询问技能的作用、触发时机以及是否应仅保留指令或包含脚本。默认为仅指令。

你还可以通过创建一个包含 `SKILL.md` 文件的文件夹手动创建技能：

```
---
name: skill-name
description: 明确说明此技能应何时及何时不应触发。
---

供 Codex 遵循的技能指令。
```

Codex 会自动检测技能更改。如果更新没有出现，请重启 Codex。

## 保存技能的位置

Codex 从存储库、用户、管理员和系统位置读取技能。对于存储库，Codex 扫描当前工作目录到存储库根目录中的每个目录下的 `.agents/skills`。如果两个技能共享相同的 `name`，Codex 不会合并它们；两者都可以出现在技能选择器中。

| 技能范围 | 位置 | 建议使用 |
| --- | --- | --- |
| `REPO` | `$CWD/.agents/skills`<br> 当前工作目录：你启动 Codex 的地方。 | 如果你在存储库或代码环境中，团队可以在相关工作文件夹中检查技能。例如，仅与微服务或模块相关的技能。 |
| `REPO` | `$CWD/../.agents/skills`<br> 启动 Codex 时，一个在当前工作目录之上的文件夹。 | 如果你在一个包含嵌套文件夹的存储库中，组织可以在父文件夹的共享区域中检查与技能相关的技能。 |
| `REPO` | `$REPO_ROOT/.agents/skills`<br> 启动 Codex 时的最上层根文件夹。 | 如果你在一个包含嵌套文件夹的存储库中，组织可以在与所有使用该存储库的人员相关的区域中检查技能。这些作为可供存储库中任何子文件夹使用的根技能。 |
| `USER` | `$HOME/.agents/skills`<br> 任何检查到用户个人文件夹中的技能。 | 用于策划与用户相关的技能，适用于用户可能处理的任何存储库。 |
| `ADMIN` | `/etc/codex/skills`<br> 在共享系统位置中检查到机器或容器中的任何技能。 | 用于 SDK 脚本、自动化，并检查每个用户在该机器上可用的默认管理员技能。 |
| `SYSTEM` | 与 Codex 一起由 OpenAI 打包。 | 对于广泛受众的有用技能，例如技能创建器和计划技能。首次启动 Codex 时可供所有人使用。 |

Codex 支持符号链接的技能文件夹，并在扫描这些位置时遵循符号链接目标。

这些位置是用于作者创作和本地发现的。当你想要将可重用技能分发到单个存储库之外，或者选择将其与应用集成一起打包时，使用 [插件](https://developers.openai.com/codex/plugins/build)。

## 使用插件分发技能

直接的技能文件夹最适合本地创作和存储库范围的工作流。如果你想要分发可重用技能，将两个或更多技能捆绑在一起，或与应用集成一起提供技能，请将它们打包成 [插件](https://developers.openai.com/codex/plugins/build)。

插件可以包括一个或多个技能。它们还可以选择性地将应用映射、MCP 服务器配置和演示资产打包在一个软件包中。

## 安装策划的技能以供本地使用

要将策划技能添加到本地 Codex 设置中以外的内置技能，请使用 `$skill-installer`。例如，要安装 `$linear` 技能：

```
$skill-installer linear
```

你还可以提示安装程序从其他存储库下载技能。Codex 会自动检测新安装的技能；如果没有出现，请重启 Codex。

使用此功能进行本地设置和实验。对于你的技能的可重用分发，优先选择插件。

## 启用或禁用技能

使用 `[[skills.config]]` 条目在 `~/.codex/config.toml` 中禁用技能，无需删除它：

```
[[skills.config]]
path = "/path/to/skill/SKILL.md"
enabled = false
```

在更改 `~/.codex/config.toml` 后重新启动 Codex。

## 可选元数据

添加 `agents/openai.yaml` 以配置 [Codex 应用](https://developers.openai.com/codex/app) 中的 UI 元数据，设置调用策略，并声明工具依赖项，以便与使用技能的体验更无缝。

```
interface:
  display_name: "可选的用户可见名称"
  short_description: "可选的用户可见描述"
  icon_small: "./assets/small-logo.svg"
  icon_large: "./assets/large-logo.png"
  brand_color: "#3B82F6"
  default_prompt: "可选的包围提示，以便使用该技能"

policy:
  allow_implicit_invocation: false

dependencies:
  tools:
    - type: "mcp"
      value: "openaiDeveloperDocs"
      description: "OpenAI 文档 MCP 服务器"
      transport: "streamable_http"
      url: "https://developers.openai.com/mcp"
```

`allow_implicit_invocation`（默认值：`true`）：当设置为 `false` 时，Codex 不会根据用户提示隐式调用技能；显式的 `$skill` 调用仍然有效。

## 最佳实践

- 每个技能应专注于一项工作。
- 除非需要确定行为或外部工具，否则优先选择指令而不是脚本。
- 以明确的输入和输出编写命令步骤。
- 测试提示与技能描述以确认正确的触发行为。

欲了解更多示例，请参见 [github.com/openai/skills](https://github.com/openai/skills) 和 [代理技能规范](https://agentskills.io/specification)。
