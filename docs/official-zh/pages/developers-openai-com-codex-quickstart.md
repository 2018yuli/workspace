## 设置

选择一个选项

应用推荐IDE扩展Codex 在你的 IDE中CLI在终端中使用CodexCloud浏览器中的Codex

Codex应用程序可在macOS和Windows上使用。

大多数Codex应用程序功能在两个平台上均可用。平台特定的例外在相关文档中注明。

1. 下载并安装Codex应用程序

下载适用于macOS或Windows的Codex应用程序。如果您使用的是基于Intel的Mac，请选择Intel版本。

[下载适用于macOS（Apple Silicon）](https://persistent.oaistatic.com/codex-app-prod/Codex.dmg) [下载适用于macOS（Intel）](https://persistent.oaistatic.com/codex-app-prod/Codex-latest-x64.dmg)

需要不同的操作系统？

[下载适用于Windows](https://get.microsoft.com/installer/download/9PLM9XGG6VKS?cid=website_cta_psi)

[获取Linux通知](https://openai.com/form/codex-app/)

2. 打开Codex并登录

下载并安装Codex应用程序后，打开它并使用您的ChatGPT帐户或OpenAI API密钥登录。

如果使用OpenAI API密钥登录，某些功能可能不可用，例如[云线程](https://developers.openai.com/codex/prompting#threads)。

3. 选择一个项目

选择一个项目文件夹，供Codex进行工作。

如果您之前使用过Codex应用程序、CLI或IDE扩展，您将看到您之前处理的项目。

4. 发送您的第一条消息

选择项目后，确保选择**本地**以让Codex在您的机器上工作，并发送您的第一条消息给Codex。

您可以问Codex关于项目或您的计算机的一切。以下是一些示例：

3. 登录并开始您的第一个任务

使用您的ChatGPT帐户或API密钥登录以开始。

Codex默认以代理模式启动，这使其能够读取文件、运行命令和在您的项目目录中进行更改。

Codex可以修改您的代码库，因此在每个任务之前和之后考虑创建Git检查点，以便您可以轻松地恢复更改。

如果您是Codex的新用户，请阅读[最佳实践指南](https://developers.openai.com/codex/learn/best-practices)。

Codex CLI在macOS、Windows和Linux上均受支持。

1. 安装Codex CLI

使用npm安装：

```
npm install -g @openai/codex
```

使用Homebrew安装：

```

```

2. 运行`codex`并登录

在终端中运行`codex`以开始使用。系统会提示您使用ChatGPT帐户或API密钥登录。

3. 请求Codex在当前目录中工作

验证身份后，您可以让Codex在当前目录中执行任务。

![](https://developers.openai.com/codex/colorcons/brain.png)告诉我关于这个项目的事。复制![](https://developers.openai.com/codex/colorcons/gamepad.png)在此仓库中构建一个经典的蛇游戏。复制![](https://developers.openai.com/codex/colorcons/search.png)以最小的、高置信度的更改找到并修复我代码库中的错误。复制

4. 使用Git检查点

Codex可以修改您的代码库，因此在每个任务之前和之后考虑创建Git检查点，以便您可以轻松地恢复更改。

如果您是Codex的新用户，请阅读[最佳实践指南](https://developers.openai.com/codex/learn/best-practices)。

[了解有关Codex CLI的更多信息](https://developers.openai.com/codex/cli)

在云中使用Codex，地址为[chatgpt.com/codex](https://chatgpt.com/codex)。

1. 在浏览器中打开Codex

访问[chatgpt.com/codex](https://chatgpt.com/codex)。您也可以通过在GitHub拉取请求评论中标记`@codex`来委派任务（需要登录ChatGPT）。

2. 设置环境

在开始您的第一个任务之前，为Codex设置环境。打开环境设置，前往[chatgpt.com/codex](https://chatgpt.com/codex/settings/environments)，并按照步骤连接GitHub仓库。

3. 启动任务并监控进度

一旦您的环境准备就绪，通过[Codex界面](https://chatgpt.com/codex)启动编码任务。您可以通过查看日志实时监控进度，或者让任务在后台运行。

![](https://developers.openai.com/codex/colorcons/brain.png)告诉我关于这个项目的事。复制![](https://developers.openai.com/codex/colorcons/brain.png)解释我应用架构的主要故障模式。复制![](https://developers.openai.com/codex/colorcons/search.png)以最小的、高置信度的更改找到并修复我代码库中的错误。复制

4. 检查更改并创建拉取请求

任务完成后，在差异视图中检查拟议的更改。您可以在结果上进行迭代，或直接在您的GitHub仓库中创建拉取请求。

Codex还提供更改的预览。您可以按原样接受PR，或在本地检出分支以测试更改：

```
git fetch
git checkout <branch-name>
```
