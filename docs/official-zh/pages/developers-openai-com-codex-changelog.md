## 新功能



  - 应用服务器集成现在支持 Unix 套接字传输、分页友好的恢复/分叉、固定环境以及远程线程配置/存储管道。 ( [#18255](https://github.com/openai/codex/pull/18255)， [#18892](https://github.com/openai/codex/pull/18892)， [#18897](https://github.com/openai/codex/pull/18897)， [#18908](https://github.com/openai/codex/pull/18908)， [#19008](https://github.com/openai/codex/pull/19008)， [#19014](https://github.com/openai/codex/pull/19014))
  - 应用服务器插件管理可以安装远程插件并升级已配置的市场。 ( [#18917](https://github.com/openai/codex/pull/18917)， [#19074](https://github.com/openai/codex/pull/19074))
  - 权限配置文件现在可以在 TUI 会话、用户轮换、MCP 沙箱状态、shell 升级和应用服务器 API 之间进行往返。 ( [#18284](https://github.com/openai/codex/pull/18284)， [#18285](https://github.com/openai/codex/pull/18285)， [#18286](https://github.com/openai/codex/pull/18286)， [#18287](https://github.com/openai/codex/pull/18287)， [#19231](https://github.com/openai/codex/pull/19231))
  - 模型提供者现在拥有模型发现，AWS/Bedrock 账户状态暴露给应用客户端。 ( [#18950](https://github.com/openai/codex/pull/18950)， [#19048](https://github.com/openai/codex/pull/19048))
  - `codex exec --json` 现在报告程序化消费者的推理令牌使用情况。 ( [#19308](https://github.com/openai/codex/pull/19308))
  - 部署跟踪现在记录工具、代码模式、会话和多代理关系，并提供调试减少命令以供检查。 ( [#18878](https://github.com/openai/codex/pull/18878)， [#18879](https://github.com/openai/codex/pull/18879)， [#18880](https://github.com/openai/codex/pull/18880))

## 错误修复

  - 中断 `/review` 和退出 TUI 不再会在代理启动或取消订阅时使界面卡死。 ( [#18921](https://github.com/openai/codex/pull/18921))
  - Exec 服务器不再在进程退出后丢失缓冲输出，现在能正确等待流关闭。 ( [#18946](https://github.com/openai/codex/pull/18946)， [#19130](https://github.com/openai/codex/pull/19130))
  - 应用服务器现在尊重明确不受信任的项目配置，而不是自动持久化信任。 ( [#18626](https://github.com/openai/codex/pull/18626))
  - WebSocket 应用服务器客户端在突发的轮次和工具输出通知期间不太可能断开连接。 ( [#19246](https://github.com/openai/codex/pull/19246))
  - Windows 沙箱启动更好地处理多个 CLI 版本和已安装的应用目录，后台 `Start-Process` 调用避免了可见的 PowerShell 窗口。 ( [#19044](https://github.com/openai/codex/pull/19044)， [#19180](https://github.com/openai/codex/pull/19180)， [#19214](https://github.com/openai/codex/pull/19214))
  - 配置/架构处理现在拒绝冲突的 MultiAgentV2 线程限制，解决相对代理角色配置路径，隐藏不支持的 MCP bearer-token 字段，并拒绝无效的 `js_repl` 图像 MIME 类型。 ( [#19129](https://github.com/openai/codex/pull/19129)， [#19261](https://github.com/openai/codex/pull/19261)， [#19294](https://github.com/openai/codex/pull/19294)， [#19292](https://github.com/openai/codex/pull/19292))

## 文档

  - 应用服务器文档和生成的架构已针对新的传输、线程、市场、固定环境和权限配置文件 API 进行了更新。 ( [#18255](https://github.com/openai/codex/pull/18255)， [#18897](https://github.com/openai/codex/pull/18897)， [#19014](https://github.com/openai/codex/pull/19014)， [#19074](https://github.com/openai/codex/pull/19074)， [#19231](https://github.com/openai/codex/pull/19231))
  - 部署跟踪文档现在涵盖调试跟踪减少工作流。 ( [#18880](https://github.com/openai/codex/pull/18880))

## 其他

  - 刷新了 `models.json` 和相关的核心、应用服务器、SDK 和 TUI 组件，以适应最新模型目录和推理默认值。 ( [#19323](https://github.com/openai/codex/pull/19323))
  - Windows Bazel CI 现在使用稳定的 PATH 和共享查询启动路径以更好地重用缓存。 ( [#19161](https://github.com/openai/codex/pull/19161)， [#19232](https://github.com/openai/codex/pull/19232))
  - 插件市场的添加/移除/启动同步内部移出了 `codex-core`，策划的插件缓存版本现在使用短 SHA。 ( [#19099](https://github.com/openai/codex/pull/19099)， [#19095](https://github.com/openai/codex/pull/19095))
  - 在 macOS 签名权限发生更改后，恢复了签名授权。 ( [#19167](https://github.com/openai/codex/pull/19167)， [#19350](https://github.com/openai/codex/pull/19350))
  - 稳定了易波动的批准弹出窗口和插件 MCP 工具发现测试。 ( [#19178](https://github.com/openai/codex/pull/19178)， [#19191](https://github.com/openai/codex/pull/19191))

#### Codex 中的 GPT-5.5



GPT-5.5 是大多数 Codex 任务的推荐选择，当您在模型选择器中看到它时。它特别适用于实施、重构、调试、测试、验证和知识工作文档。



要切换到 GPT-5.5：



  - 在 CLI 中，使用以下命令开始一个新线程：


    ```
    codex --model gpt-5.5


    ```



    或在会话中使用 `/model`。
  - 在 IDE 扩展中，从创作器的模型选择器中选择 GPT-5.5。
  - 在 Codex 应用中，从创作器的模型选择器中选择 GPT-5.5。

如果您尚未看到 GPT-5.5，请将 CLI、IDE 扩展或 Codex 应用更新为最新版本。在推出期间，如果 GPT-5.5 尚不可用，请继续使用 GPT-5.4。



#### Codex 应用中的浏览器使用

Codex 应用现在可以让 Codex 操作应用内浏览器，用于本地开发服务器和文件支持页面。当 Codex 需要单击渲染的 UI、重现视觉错误或验证应用程序内部的本地修复时，可以请求 Codex 使用浏览器。

浏览器使用通过捆绑的浏览器插件运行。在设置中，您可以管理插件并查看允许或阻止的网站。



#### 自动批准审核

Codex 可以通过自动审核代理将符合条件的批准提示路由，审核请求运行之前进行。当配置时，Codex 应用显示一个自动审核项，显示审核状态和风险级别，因此您可以看到审核者是否批准、拒绝、已停止或在决定之前超时。

## 新功能



  - TUI 现在具有快速推理控制: `Alt+,` 降低推理，`Alt+.` 提高推理，已接受的模型升级现在将推理重置为新模型的默认，而不是保留过时的设置。 ( [#18866](https://github.com/openai/codex/pull/18866)， [#19085](https://github.com/openai/codex/pull/19085))
  - 应用服务器会话现在可以管理多个环境，并为每个轮次选择一个环境和工作目录，这使得多工作区和远程设置的精确定位变得更容易。 ( [#18401](https://github.com/openai/codex/pull/18401)， [#18416](https://github.com/openai/codex/pull/18416))
  - 为 OpenAI 兼容提供商添加了一流的 Amazon Bedrock 支持，包括 AWS SigV4 签名和基于 AWS 凭证的身份验证。 ( [#17820](https://github.com/openai/codex/pull/17820))
  - 远程插件市场现在可以直接列出和读取，详细查找更加可靠，结果页面更大。 ( [#18452](https://github.com/openai/codex/pull/18452)， [#19079](https://github.com/openai/codex/pull/19079))
  - 钩子现在是稳定的，可以在 `config.toml` 中内联配置并在管理的 `requirements.toml` 中管理，能够观察 MCP 工具以及 `apply_patch` 和长运行的 Bash 会话。 ( [#18893](https://github.com/openai/codex/pull/18893)， [#18385](https://github.com/openai/codex/pull/18385)， [#18391](https://github.com/openai/codex/pull/18391)， [#18888](https://github.com/openai/codex/pull/18888)， [#19012](https://github.com/openai/codex/pull/19012))
  - 符合条件的 ChatGPT 计划现在默认使用快速服务级别，除非您明确选择退出。 ( [#19053](https://github.com/openai/codex/pull/19053))

## 错误修复

  - 在获准的 ChatGPT 主机之间保持 Cloudflare cookies，减少 HTTP 基于 ChatGPT 流中的身份验证中断。 ( [#17783](https://github.com/openai/codex/pull/17783))
  - 修复了远程应用服务器可靠性问题，使得在负载下 WebSocket 事件持续流动，关闭时在远程工作者清理过程中不再发生失败。 ( [#18932](https://github.com/openai/codex/pull/18932)， [#18936](https://github.com/openai/codex/pull/18936))
  - 修复了权限模式漂移，以便 `/permissions` 的变化在内部对话中生存，更新后 Full Access 状态正确反映在 MCP 审批处理中。 ( [#18924](https://github.com/openai/codex/pull/18924)， [#19033](https://github.com/openai/codex/pull/19033))
  - 修复了 `wait_agent`，在邮箱工作已经排队时可以立即返回，而不是等待新通知或超时。 ( [#18968](https://github.com/openai/codex/pull/18968))
  - 修复了相对命令的本地 stdio MCP 启动，确保 `cwd` 下的基本路径解析修复，与 CLI 行为一致。 ( [#19031](https://github.com/openai/codex/pull/19031))
  - 启动在托管配置边缘案例中失败的频率降低：未知特性需求警告而非中止，云需求错误更清晰地说明失败原因。 ( [#19038](https://github.com/openai/codex/pull/19038)， [#19078](https://github.com/openai/codex/pull/19078))
