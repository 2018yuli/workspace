在 Slack 中使用 Codex 从频道和线程启动编码任务。提到 `@Codex` 并提供一个提示，Codex 将创建一个云任务并回复结果。

![Codex Slack 集成示例](https://developers.openai.com/images/codex/integrations/slack-example.png)

## 设置 Slack 应用

1. 设置 [Codex 云任务](https://developers.openai.com/codex/cloud)。您需要一个 Plus、Pro、Business、Enterprise 或 Edu 计划（请参见 [ChatGPT 定价](https://chatgpt.com/pricing)）、一个连接的 GitHub 账户，以及至少一个 [环境](https://developers.openai.com/codex/cloud/environments)。
2. 转到 [Codex 设置](https://chatgpt.com/codex/settings/connectors) 并为您的工作区安装 Slack 应用。根据您的 Slack 工作区政策，可能需要管理员批准安装。
3. 在一个频道中添加 `@Codex`。如果您尚未添加，Slack 会在您提到它时提示您。

## 开始任务

1. 在一个频道或线程中，提到 `@Codex` 并包含您的提示。Codex 可以引用线程中早期的消息，因此您通常不需要重申上下文。
2. （可选）在您的提示中指定一个环境或存储库，例如：`@Codex 修复上述内容在 openai/codex`。
3. 等待 Codex 反应 (👀) 并回复任务链接。当它完成时，Codex 会发布结果，并根据您的设置在线程中给出答案。

### Codex 如何选择环境和仓库

- Codex 会审查您可以访问的环境，并选择最匹配您请求的环境。如果请求模糊，它会回退到您最近使用的环境。
- 任务在该环境的仓库映射中列出的第一个仓库的默认分支上运行。如果您需要不同的默认分支或更多仓库，请在 Codex 中更新仓库映射。
- 如果没有合适的环境或仓库可用，Codex 将在 Slack 中回复说明如何解决问题，然后重试。

### 企业数据控制

默认情况下，Codex 会在线程中回复答案，其中可能包含它运行的环境中的信息。
为了防止这种情况，企业管理员可以在 [ChatGPT 工作区设置](https://chatgpt.com/admin/settings) 中清除 **允许 Codex Slack 应用在任务完成时发布答案** 的选项。当管理员关闭答案时，Codex 只会回复任务链接。

### 数据使用、隐私和安全

当您提到 `@Codex` 时，Codex 收到您的消息和线程历史记录，以理解您的请求并创建任务。
数据处理遵循 OpenAI 的 [隐私政策](https://openai.com/privacy)、[使用条款](https://openai.com/terms/) 和其他适用 [政策](https://openai.com/policies)。
有关安全性的更多信息，请参见 Codex [安全文档](https://developers.openai.com/codex/agent-approvals-security)。

Codex 使用大型语言模型，可能会出错。请务必审核答案和差异。

### 提示和故障排除

- **缺少连接**：如果 Codex 无法确认您的 Slack 或 GitHub 连接，则会回复一个链接以重新连接。
- **意外的环境选择**：在线程中回复您希望使用的环境（例如，`请在 openai/openai 中运行此项（已应用）`），然后再次提到 `@Codex`。
- **长或复杂的线程**：在您最新的消息中总结关键细节，以便 Codex 不会错过埋藏在早期线程中的上下文。
- **工作区发帖**：某些企业工作区限制发布最终答案。在这种情况下，打开任务链接以查看进度和结果。
- **更多帮助**：请参见 [OpenAI 帮助中心](https://help.openai.com/)。
