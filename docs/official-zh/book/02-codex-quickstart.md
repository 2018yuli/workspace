# 第02章 快速入门

> 原始页面：[Quickstart – Codex | OpenAI Developers](https://developers.openai.com/codex/quickstart)

## 本章目标
这章面向会 Python、懂一点 LangChain、但还没有真正把 AI Agent 当成工程系统来使用的读者。重点不是背术语，而是把“这个功能为什么存在、什么时候该用、怎么安全地用”讲清楚。

## 先用一个高中数学类比
把这一章看成在学习一个新的数学对象。先认识它的定义，再看它的性质，最后看它怎么解题。

## 严谨定义
严格地说，这一章讨论的是 Codex 体系中的一个功能模块，以及它与输入、状态、输出之间的关系。

## 你读完应该抓住什么
- 每个ChatGPT计划都包括Codex。
- `设置`：选择一个选项

## 分步理解
### 正文
先说直白版：
每个ChatGPT计划都包括Codex。
您还可以通过使用OpenAI API密钥登录来使用Codex和API积分。

把它理解成一个更严谨的过程：
1. 每个ChatGPT计划都包括Codex。
2. 您还可以通过使用OpenAI API密钥登录来使用Codex和API积分。

### 设置
先说直白版：
选择一个选项
应用推荐IDE扩展Codex 在你的 IDE中CLI在终端中使用CodexCloud浏览器中的Codex
Codex应用程序可在macOS和Windows上使用。
大多数Codex应用程序功能在两个平台上均可用。平台特定的例外在相关文档中注明。

把它理解成一个更严谨的过程：
1. 选择一个选项
2. 应用推荐IDE扩展Codex 在你的 IDE中CLI在终端中使用CodexCloud浏览器中的Codex
3. Codex应用程序可在macOS和Windows上使用。
4. 大多数Codex应用程序功能在两个平台上均可用。平台特定的例外在相关文档中注明。

## 实战视角
如果你会 Python，可以把这一章里的能力理解成一个可以读文件、调工具、保留状态、按规则执行的程序代理。它和 LangChain 的链、工具、记忆很像，但 Codex 更强调真实工程环境：仓库、命令、审阅、权限、并行任务。

## 给高中水平读者的最后一句话
不要急着一次学完整个体系。把每章都当成“定义 + 例子 + 适用边界”三件事去掌握，AI Agent 就会从模糊概念变成你能实际操作的工程对象。
