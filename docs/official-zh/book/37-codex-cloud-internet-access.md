### 代理互联网访问的风险
先说直白版：
启用代理互联网访问会增加安全风险，包括：
- 来自不可信网站内容的提示注入
- 代码或秘密的外泄
- 下载恶意软件或易受攻击的依赖项

把它理解成一个更严谨的过程：
1. 启用代理互联网访问会增加安全风险，包括：
2. 来自不可信网站内容的提示注入
3. 代码或秘密的外泄
4. 下载恶意软件或易受攻击的依赖项

### 配置代理互联网访问
先说直白版：
代理互联网访问是按环境进行配置的。
- **关闭**: 完全阻止互联网访问。
- **打开**: 允许互联网访问，您可以通过域名白名单和允许的 HTTP 方法进行限制。

把它理解成一个更严谨的过程：
1. 代理互联网访问是按环境进行配置的。
2. **关闭**: 完全阻止互联网访问。
3. **打开**: 允许互联网访问，您可以通过域名白名单和允许的 HTTP 方法进行限制。

### 域名白名单
先说直白版：
您可以选择预设的白名单：
- **无**: 使用空白名单并从头开始指定域名。
- **常见依赖项**: 使用预设的域名白名单，这些域名通常用于下载和构建依赖项。请参阅 常见依赖项 中的列表。
- **全部（无限制）**: 允许所有域名。

把它理解成一个更严谨的过程：
1. 您可以选择预设的白名单：
2. **无**: 使用空白名单并从头开始指定域名。
3. **常见依赖项**: 使用预设的域名白名单，这些域名通常用于下载和构建依赖项。请参阅 常见依赖项 中的列表。
4. **全部（无限制）**: 允许所有域名。

### 允许的 HTTP 方法
先说直白版：
为了额外保护，将网络请求限制为 `GET`，`HEAD` 和 `OPTIONS`。使用其他方法（`POST`、`PUT`、`PATCH`、`DELETE` 等）的请求将被阻止。
