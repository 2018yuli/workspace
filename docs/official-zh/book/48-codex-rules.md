### 理解规则字段
先说直白版：
`prefix_rule()`支持以下字段：
- `pattern` **（必需）**：一个非空列表，定义要匹配的命令前缀。每个元素可以是：
- 一个字面字符串（例如，`"pr"`）。
- 字面量的联合（例如，`["view", "list"]`），以匹配该参数位置的替代项。

把它理解成一个更严谨的过程：
1. `prefix_rule()`支持以下字段：
2. `pattern` **（必需）**：一个非空列表，定义要匹配的命令前缀。每个元素可以是：

4. 字面量的联合（例如，`["view", "list"]`），以匹配该参数位置的替代项。

### 当 Codex 可以安全地拆分脚本
先说直白版：
如果 shell 脚本是由以下构成的线性命令链：
- 普通单词（没有变量扩展，没有 `VAR=...`、`$FOO`、`*` 等）
- 通过安全操作符连接（`&&`、`||`、`;` 或 `|`）
那么 Codex 会解析它（使用 tree-sitter）并在应用规则之前将其拆分为单独的命令。

## 实战视角
如果你会 Python，可以把这一章里的能力理解成一个可以读文件、调工具、保留状态、按规则执行的程序代理。它和 LangChain 的链、工具、记忆很像，但 Codex 更强调真实工程环境：仓库、命令、审阅、权限、并行任务。
