### 跟踪 Codex 使用情况的方法
先说直白版：
有三种方法可以监控 Codex 使用情况，具体取决于你的需求：
- **分析仪表板**：快速查看采用情况和代码审查影响。
- **分析 API**：将结构化的日常指标导入到你的数据仓库或 BI 工具中。
- **合规 API**：导出详细的活动日志以进行审核、监控和调查。

把它理解成一个更严谨的过程：
1. 有三种方法可以监控 Codex 使用情况，具体取决于你的需求：
2. **分析仪表板**：快速查看采用情况和代码审查影响。
3. **分析 API**：将结构化的日常指标导入到你的数据仓库或 BI 工具中。
4. **合规 API**：导出详细的活动日志以进行审核、监控和调查。

## 实战视角
如果你会 Python，可以把这一章里的能力理解成一个可以读文件、调工具、保留状态、按规则执行的程序代理。它和 LangChain 的链、工具、记忆很像，但 Codex 更强调真实工程环境：仓库、命令、审阅、权限、并行任务。

## 给高中水平读者的最后一句话
不要急着一次学完整个体系。把每章都当成“定义 + 例子 + 适用边界”三件事去掌握，AI Agent 就会从模糊概念变成你能实际操作的工程对象。
