### 配置优先级
先说直白版：
Codex 按以下顺序解析值（优先级最高的在前）：
1. CLI 标志和 `--config` 覆盖 2. 配置文件 值（来自 `--profile <name>`） 3. 项目配置文件：`.codex/config.toml`，从项目根向下到您当前的工作目录（离得近者优先；仅限受信任项目） 4. 用户配置：`~/.codex/config.toml` 5. 系统配置（如果存在）：Unix 上的 `/etc/codex/config.toml` 6. 内置默认值
使用该优先级在顶层设置共享默认值，并保持配置文件集中于不同值。
如果您将项目标记为不受信任，Codex 将跳过项目范围 `.codex/` 层，包括项目本地配置、钩子和规则。用户和系统配置仍将加载，包括用户/全局钩子和规则。

## 实战视角
如果你会 Python，可以把这一章里的能力理解成一个可以读文件、调工具、保留状态、按规则执行的程序代理。它和 LangChain 的链、工具、记忆很像，但 Codex 更强调真实工程环境：仓库、命令、审阅、权限、并行任务。
