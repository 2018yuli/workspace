# 第71章 功能成熟度

> 原始页面：[Feature Maturity – Codex | OpenAI Developers](https://developers.openai.com/codex/feature-maturity)

## 本章目标
这章面向会 Python、懂一点 LangChain、但还没有真正把 AI Agent 当成工程系统来使用的读者。重点不是背术语，而是把“这个功能为什么存在、什么时候该用、怎么安全地用”讲清楚。

## 先用一个高中数学类比
配置像给函数预先设定参数。公式不变，但参数不同，图像和输出会明显不同。

## 严谨定义
严格地说，配置是运行时行为的参数化描述。

## 你读完应该抓住什么
- 一些 Codex 功能以成熟度标签发布，以便您了解每个功能的可靠性、可能的变化以及期望的支持级别。

## 分步理解
### 正文
先说直白版：
一些 Codex 功能以成熟度标签发布，以便您了解每个功能的可靠性、可能的变化以及期望的支持级别。
| 成熟度 | 意思 | 指导 | | --- | --- | --- | | 正在开发中 | 尚未准备好使用。 | 不要使用。 | | 实验性 | 不稳定，OpenAI 可能会移除或更改它。 | 自行承担风险使用。 | | 测试版 | 准备进行广泛测试；在大多数方面完成，但某些方面可能会根据用户反馈而改变。 | 适合大多数评估和试点；预计会有小的变化。 | | 稳定 | 完全支持、记录，并准备广泛使用；行为和配置随时间保持一致。 | 适合生产使用；删除通常通过弃用流程进行。 |

## 实战视角
如果你会 Python，可以把这一章里的能力理解成一个可以读文件、调工具、保留状态、按规则执行的程序代理。它和 LangChain 的链、工具、记忆很像，但 Codex 更强调真实工程环境：仓库、命令、审阅、权限、并行任务。

## 给高中水平读者的最后一句话
不要急着一次学完整个体系。把每章都当成“定义 + 例子 + 适用边界”三件事去掌握，AI Agent 就会从模糊概念变成你能实际操作的工程对象。
