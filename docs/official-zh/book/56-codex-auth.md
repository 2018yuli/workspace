## 分步理解
### OpenAI 身份验证
先说直白版：
Codex 支持在使用 OpenAI 模型时，通过两种方式登录：
- 使用 ChatGPT 登录以获得订阅访问权限
- 使用 API 密钥登录以获得基于使用量的访问权限
Codex 云要求使用 ChatGPT 登录。Codex CLI 和 IDE 扩展支持两种登录方式。

把它理解成一个更严谨的过程：
1. Codex 支持在使用 OpenAI 模型时，通过两种方式登录：
2. 使用 ChatGPT 登录以获得订阅访问权限
3. 使用 API 密钥登录以获得基于使用量的访问权限
4. Codex 云要求使用 ChatGPT 登录。Codex CLI 和 IDE 扩展支持两种登录方式。

### 使用 ChatGPT 登录
先说直白版：
当您从 Codex 应用、CLI 或 IDE 扩展使用 ChatGPT 登录时，Codex 会为您打开一个浏览器窗口以完成登录流程。登录后，浏览器会将访问令牌返回给 CLI 或 IDE 扩展。

把它理解成一个更严谨的过程：
1. 当您从 Codex 应用、CLI 或 IDE 扩展使用 ChatGPT 登录时，Codex 会为您打开一个浏览器窗口以完成登录流程。登录后，浏览器会将访问令牌返回给 CLI 或 IDE 扩展。

### 使用 API 密钥登录
先说直白版：
您还可以使用 API 密钥登录 Codex 应用、CLI 或 IDE 扩展。请从 OpenAI 仪表板 获取您的 API 密钥。
OpenAI 会通过您的 OpenAI 平台账户按标准 API 费率对 API 密钥的使用进行计费。请查看 API 定价页面。
依赖于 ChatGPT 余额的功能，例如 快速模式，仅在使用 ChatGPT 登录时可用。如果您使用 API 密钥登录，Codex 将使用标准 API 定价。
推荐在程序化的 Codex CLI 工作流中使用 API 密钥身份验证（例如 CI/CD 作业）。请勿在不受信任或公共环境中暴露 Codex 执行。

### 保护您的 Codex 云账户
先说直白版：
Codex 云直接与您的代码库交互，因此需要比许多其他 ChatGPT 功能更强的安全性。启用多因素身份验证 (MFA)。
如果您使用社交登录提供商（谷歌、微软、苹果），则不需要在您的 ChatGPT 账户中启用 MFA，但您可以与您的社交登录提供商一起设置。

### 登录缓存
先说直白版：
当您使用 ChatGPT 或 API 密钥登录 Codex 应用、CLI 或 IDE 扩展时，Codex 会缓存您的登录详情，并在您下次启动 CLI 或扩展时重用它们。CLI 和扩展共享相同的缓存登录详情。如果您从其中一个注销，则下次启动 CLI 或扩展时需要重新登录。
Codex 在 `~/.codex/auth.json` 的明文文件中或在您操作系统特定的凭证存储中本地缓存登录详情。
对于使用 ChatGPT 会话的登录，Codex 会在使用期间在令牌过期前自动刷新令牌，因此活跃会话通常会持续进行，而无需再次浏览器登录。

把它理解成一个更严谨的过程：
1. 当您使用 ChatGPT 或 API 密钥登录 Codex 应用、CLI 或 IDE 扩展时，Codex 会缓存您的登录详情，并在您下次启动 CLI 或扩展时重用它们。CLI 和扩展共享相同的缓存登录详情。如果您从其中一个注销，则下次启动 CLI 或扩展时需要重新登录。

## 实战视角
如果你会 Python，可以把这一章里的能力理解成一个可以读文件、调工具、保留状态、按规则执行的程序代理。它和 LangChain 的链、工具、记忆很像，但 Codex 更强调真实工程环境：仓库、命令、审阅、权限、并行任务。
